CUPS

Common Unix Printing System

none

Instalación y verificación

En el equipo donde se podrá administrar el servicio instalaremos los paquetes necesarios:

apt install -y cups cups-pdf

Para verificar el estado del servicio ejecutaremos el siguiente comando:

systemctl status cups

A continuación permitiremos el acceso a cualquier red al servicio:

cupsctl --remote-any

O podemos limitar el acceso a solo la red local configurando el siguiente archivo:

nano /etc/cups/cupsd.conf

Buscaremos una sección <Location /> y la dejaremos así, pero cambiando 192.168.1.* por la dirección de la red local:

<Location />
  Order allow,deny
  Allow localhost
  Allow 192.168.1.*
</Location>

Luego aplicaremos los cambios reiniciando el servicio:

systemctl restart cups

Ahora verificaremos que el servicio cups esté escuchando en los puertos correspondientes. Debería ser el puerto 631:

lsof -i -P -n | grep cups

Administración de la interfaz gráfica

Primero tendremos que entrar al panel de administración web de cups. La URL debería ser:

https://<IP_Local_Servidor>:631/admin

En la sección de Printers pulsaremos Add Printer.

- Seleccionaremos CUPS-PDF (Virtual PDF Printer) y luego Continue.

- Definiremos un nombre a la impresora, una descripción y seleccionaremos Share this printer. Luego pulsaremos Continue.

- Ahora seleccionaremos en la sección Make el driver Generic, en la sección Model el modelo Generic CUPS-PDF Printer (w/ options) (en) y por último pulsaremos Add Printer.

Ahora nos dirigiremos a la sección de impresoras del panel. La URL debería ser:

https://<IP_Local_Servidor>:631/printers

Seleccionaremos la impresora que se ha creado en el paso anterior y ya podremos ver la cola de impresión.

Después de crear la impresora en el panel web (CUPS-PDF), hay que activarla y permitir trabajos especificando el nombre configurado de la impresora:

cupsenable <Nombre_Impresora>
cupsaccept <Nombre_Impresora>

Ahora añadiremos una política al servicio mediante el siguiente fichero:

nano /etc/cups/cupsd.conf

Buscaremos una sección <Policy default> y configuraremos la siguiente línea:

  JobPrivateValues none

Luego reiniciaremos el servicio para aplicar los cambios:

systemctl restart cups

Agregar la impresora al Sistema

En un equipo de la red listaremos las impresoras que se detecten:

lpinfo -v

Ahora añadiremos una impresora al sistema local asignándole un nombre (-p) mediante la IP del equipo (ipp://...) que creó la impresora y el nombre que le asignó a esta (printers/...):

lpadmin -p <Nombre_Local> -E -v ipp://<IP_Servidor>:631/printers/<Nombre_Impresora> -m everywhere

Una vez añadida la activaremos y permitiremos trabajos de impresión:

cupsenable <Nombre_Local>
cupsaccept <Nombre_Local>

Verificaremos que la impresora esté en el sistema:

lpstat -p -d

Nota: La impresora puede aparecer inactiva ya que es una impresora virtual sin ningún dispositivo físico conectado.

Para verificar que se pueda imprimir podemos crear un fichero de prueba y enviarlo a la impresora:

echo "Contenido de prueba" > <Fichero_de_Prueba>
lp -d <Nombre_Local> <Fichero_de_Prueba>

Servidor: En la página de impresoras deberíamos ver los trabajos enviados:

https://<IP_Local_Servidor>:631/printers

Los PDFs generados se guardarán normalmente en la carpeta del servidor: /var/spool/cups-pdf/.
