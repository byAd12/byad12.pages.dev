¿Qué es XPATH?

Xpath permite navegar entre los elementos y atributos de un documento XML. Para hacerlo, se basa en las relaciones de parentesco entre los nodos del documento.

Se creó para ser utilizado en XSLT pero también se utiliza en XML Schema, XQuery, XLink, XPointer y XForms.

Tiene varias versiones como la 1.0 que es la primera versión y tiene menos opciones, u otras versiones como la 3.1, que es la última.

XPath se usa definiendo expresiones de camino (paths) para seleccionar nodos o conjuntos de nodos en un documento XML, asumiendo que su estructura interna es la de un árbol. Al aplicar una expresión, se obtiene un conjunto de nodos (que puede ser vacío).

Definición de Nodos

XPath trata los documentos XML como un árbol de nodos. Dentro de este árbol, podemos diferenciar diferentes tipos de nodos según su localización en el documento:

  • Nodo raíz - siempre es /
    • El elemento raíz del documento XML no es lo mismo que el nodo raíz de XPATH.
    • No puede tener nodos padre.
    • Puede tener cero o más nodos hijo.
    • Los nodos hijo puede contener nodos de elemento, de texto, de instrucción o de comentario.
    • Para ser un documento bien formado, el nodo de documento debe tener exactamente un nodo elemento hijo y ningún nodo texto hijo.

  • Nodo elemento - sub-etiquetas del nodo raíz u otro nodo elemento
  • Nodo atributo - el atributo que pueda tener un nodo elemento
  • Nodo de texto - el contenido de un nodo elemento
  • Nodo de comentario - un comentario con formato <!-- ... -->

Estructura de un path

Un path se define separando el nombre de los nodos con barras - /, como por ejemplo:

/tienda/productos/producto/nombre/text()

En el ejemplo podemos identificar el nodo raíz - /, seguido de una serie de nodos de elementos hasta el nodo nombre, a la cual se le aplica una función para obtener un nodo de texto.

El documento XML se podría ver por ejemplo así:

<?xml version="1.0" encoding="utf-8" standalone="no"?>
<tienda>
    <productos>
        <producto id="p01">
            <nombre>Agua</nombre>
        </producto>
    </productos>
</tienda>

Y como resultado cogería todos los nodos de texto dentro de todos los nodos elementos nombre.

Hay que tener en cuenta que se puede definir con rutas absolutas o relativas:

En las rutas absolutas se escribe cada nodo hasta llegar al resultado empezando desde la raíz, como en el ejemplo anterior.

En las rutas relativas se escribe // para llegar a un nodo en específico, como por ejemplo:

//nombre/text()

En este ejemplo saltamos directamente a todos los nodos elementos llamados nombre, pero hay que tener cuidado ya que si hay varios nodos con el mismo nombre en rutas diferentes se van a coger todos.

Uso del eje "child" (Hijos)

En XPath, los nodos hijo son los elementos que están directamente dentro de otro. Por defecto, cualquier ruta que escribas usa el eje child.

Si queremos usar la sintaxis explícita para el mismo documento del servidor:

/servidor/child::servicios/child::servicio

Esto es equivalente a escribir /servidor/servicios/servicio. Se usa para indicar que buscamos solo en los hijos directos.

Uso de la función "string()"

A diferencia de text(), que solo toma el texto de un nodo específico, la función string() devuelve la concatenación de todo el texto del nodo y sus descendientes.

Tomando como ejemplo este nodo raíz:

<producto>
    <nombre>Pantalla</nombre>
    <precio>19.99</precio>
</producto>

Si aplicamos string() al nodo raíz:

/producto/string()

El resultado sería "Pantalla19.99". Es muy útil cuando necesitas extraer todo el contenido legible de una etiqueta que tiene sub-etiquetas dentro.

Selección de un nodo elemento

Primero llegaremos a la ruta donde se situe el nodo elemento tiempo_ejecucion del siguiente documento XML:

<?xml version="1.0" encoding="utf-8" standalone="no"?>
<servidor>
    <servicios>
        <servicio programa="bind9">
            <tiempo_ejecucion unidad="segundos">135</tiempo_ejecucion>
        </servicio>
    </servicios>
</servidor>

Si queremos obtener la etiqueta completa con todo su contenido simplemente lo dejaremos así:

/servidor/servicios/servicio/tiempo_ejecucion

En cambio, si solo queremos obtener el nodo texto dentro de ese nodo elemento, que en este caso es "135", tenedremos que hacer:

/servidor/servicios/servicio/tiempo_ejecucion/text()

La función parent() devolverá el nombre del nodo que contiene el nodo elemento seleccionado:

/servidor/servicios/servicio/tiempo_ejecucion/parent()

Según el ejemplo devolverá "<servicio programa="bind9">" pero si seleccionamos el nodo texto devolverá "<tiempo_ejecucion unidad="segundos">".

Selección de un nodo atributo

Primero llegaremos a la ruta donde se situe el nodo atributo unidad del siguiente documento XML:

<?xml version="1.0" encoding="utf-8" standalone="no"?>
<servidor>
    <servicios>
        <servicio programa="bind9">
            <tiempo_ejecucion unidad="segundos">135</tiempo_ejecucion>
        </servicio>
    </servicios>
</servidor>

Para seleccionar el nodo atributo pondremos el símbolo @ antes del nombre del atributo.

Según versiones modernas de XPATH tendremos que definir que vamos a coger el contenido dentro del atributo con string(), lo que nos devolverá "segundos":

/servidor/servicios/servicio/tiempo_ejecucion/@unidad/string()

La función node() devolverá el nombre del atributo seleccionado:

/servidor/servicios/servicio/@programa/node()

Según el ejemplo devolverá "programa".

La función parent() devolverá el nombre del nodo que contiene el atributo seleccionado:

/servidor/servicios/servicio/@programa/parent()

Según el ejemplo devolverá "<servicio>".

La función type() devolverá en formato xs:[TIPO] el tipo de valor que contiene el atributo:

/servidor/servicios/servicio/@programa/type()

Según el ejemplo devolverá "xs:untypedAtomic".

Filtro por coincidencias

Podemos filtrar los servicios del siguiente XML que cumplan una condición:

<?xml version="1.0" encoding="utf-8" standalone="no"?>
<servidor>
    <servicios>
        <servicio programa="bind9">
            <tipo_proceso>Padre</tipo_proceso>
            <tiempo_ejecucion unidad="segundos">135</tiempo_ejecucion>
        </servicio>
        <servicio programa="apache">
            <tipo_proceso>Hijo</tipo_proceso>
            <tiempo_ejecucion unidad="minutos">312</tiempo_ejecucion>
        </servicio>
    </servicios>
</servidor>

Podemos filtrar los servicios cuyo programa sea exactamente "bind9":

/servidor/servicios/servicio[@programa = "bind9"]

Según el ejemplo devolverá:

<servicio programa="bind9">
    <tipo_proceso>Padre</tipo_proceso>
    <tiempo_ejecucion unidad="segundos">135</tiempo_ejecucion>
</servicio>

Podemos filtrar los servicios cuyo contenido de texto no sea exactamente "bind9":

/servidor/servicios/servicio[@programa != "bind9"]

Según el ejemplo devolverá:

<servicio programa="apache">
    <tipo_proceso>Hijo</tipo_proceso>
    <tiempo_ejecucion unidad="minutos">312</tiempo_ejecucion>
</servicio>

Los ejemplos anteriores devolverán todo el nodo servicio incluido todo los nodos que contenga.

También podemos definir que nos devuelva solo un nodo específico de los servicios que cumplan la condición, como por ejemplo el nodo tipo_proceso:

/servidor/servicios/servicio[@programa != "bind9"]/tipo_proceso/text()

Según el ejemplo devolverá "Hijo".

Filtro de números

Podemos filtrar los servicios del siguiente XML que cumplan una condición sobre un número:

<?xml version="1.0" encoding="utf-8" standalone="no"?>
<servidor>
    <servicios>
        <servicio programa="bind9">
            <tipo_proceso>Padre</tipo_proceso>
            <tiempo_ejecucion unidad="segundos">135</tiempo_ejecucion>
        </servicio>
        <servicio programa="apache">
            <tipo_proceso>Hijo</tipo_proceso>
            <tiempo_ejecucion unidad="minutos">312</tiempo_ejecucion>
        </servicio>
    </servicios>
</servidor>

Filtramos los tiempos de ejecución si son mayores a 100:

/servidor/servicios/servicio[tiempo_ejecucion/text() > "100"]

Filtramos los tiempos de ejecución si son menores a 100:

/servidor/servicios/servicio[tiempo_ejecucion/text() < "100"]

Filtramos los tiempos de ejecución si son mayores o iguales a 100:

/servidor/servicios/servicio[tiempo_ejecucion/text() >= "100"]

Filtramos los tiempos de ejecución si son menores o iguales a 100:

/servidor/servicios/servicio[tiempo_ejecucion/text() <= "100"]

Los ejemplos anteriores devolverán todo el nodo servicio incluido todo los nodos que contenga.

Ejes de búsqueda (Axes)

Los ejes permiten localizar nodos en relación al nodo actual, permitiendo una navegación más compleja que las rutas simples.

  • ancestor: Selecciona todos los antepasados (padre, abuelo, etc.) del nodo actual.
  • descendant: Selecciona todos los descendientes (hijos, nietos, etc.).
  • following-sibling: Selecciona todos los hermanos que aparecen después del nodo actual.
  • preceding-sibling: Selecciona todos los hermanos que aparecen antes del nodo actual.
  • self: Selecciona el nodo actual (se puede abreviar con un punto .).

Ejemplo para obtener el nombre del "padre" de un nodo específico:

//tiempo_ejecucion/parent::node()

Uso de Comodines (Wildcards)

XPath utiliza comodines que nos permiten buscar elementos sin conocer su nombre exacto.

  • * - Selecciona todos los nodos elemento.
  • @* - Selecciona todos los atributos de un nodo.
  • node() - Selecciona cualquier tipo de nodo (elemento, texto, comentario, etc.).

Selecciona todos los elementos directos dentro de servicios:

/servidor/servicios/*

Selecciona todos los elementos directos dentro de los nodos dentro de servicios:

/servidor/servicios/*/*

Ítems y Tipos de Datos

En XPath, la evaluación de una expresión siempre da como resultado una secuencia de Ítems. Un ítem puede ser de dos tipos:

  • Nodos: Cualquier nodo del árbol XML (raíz, elemento, atributo, etc.).
  • Valores atómicos: Valores simples como cadenas de texto, números o booleanos (true/false).

Además, XPath maneja principalmente cuatro tipos de datos básicos: Conjunto de nodos, Booleano, Cadena de caracteres (String) y Numérico.

Acceso a datos de otro documento XML

XPath permite hacer consultas no solo sobre el documento actual, sino también extraer información de archivos externos utilizando la función doc().

Si por ejemplo queremos acceder al nodo raíz de un archivo llamado "config.xml" que está en la misma carpeta:

doc("config.xml")/servidor

Localizar comentarios

Si necesitamos extraer los comentarios (las etiquetas que tienen el formato <!-- ... -->) de un documento XML, usamos la función de prueba de nodo comment().

Para localizar todos los comentarios dentro del nodo raíz:

/servidor/comment()

Unión de Nodos y Secuencias

En XPath, a menudo necesitamos seleccionar varios elementos distintos al mismo tiempo. Para ello, tenemos dos operadores clave que no debemos confundir:

1. Operador de Unión ( | )

Se representa con la barra vertical |. Une dos o más conjuntos de nodos. Elimina automáticamente los duplicados y devuelve los nodos en el mismo orden en el que aparecen en el documento XML.

Seleccionar todos los programas y todos los tiempos de ejecución a la vez:

//servicio/@programa | //tiempo_ejecucion

2. Operador de Secuencia ( , )

Se representa con una coma , (disponible en XPath 2.0 y 3.1). Permite concatenar resultados formando una lista o secuencia. A diferencia de la unión, no elimina duplicados y respeta el orden en el que escribes la consulta.

Crear una secuencia que devuelva primero los tiempos y luego los programas:

(//tiempo_ejecucion , //servicio/@programa)

Funciones avanzadas útiles

Existen funciones adicionales muy potentes para tratar datos más complejos o listas largas:

Función Descripción y Ejemplo
distinct-values() Elimina los elementos duplicados de una secuencia.
distinct-values(//tipo_proceso)
avg() Devuelve el valor promedio de una secuencia de números.
avg(//tiempo_ejecucion)
replace() Reemplaza una cadena de texto por otra. Usa 3 valores: nodo, texto a buscar, texto nuevo.
replace(., "Padre", "Principal")
matches() Comprueba si una cadena coincide con una expresión regular (Regex). Devuelve true/false.
matches(., "[0-9]+")
