Debian PXE

Automatizar la instalación de Debian

none

1. Configuración del servidor DHCP

1.1. Instalaremos el servicio correspondiente en el servidor:

apt-get update
apt-get install -y isc-dhcp-server

1.2. A continuación deberemos modificar el siguiente fichero para configurar las interfaces de red:

nano /etc/default/isc-dhcp-server

Dentro de este fichero reemplazaremos las siguientes líneas:

INTERFACESv4=""
INTERFACESv6=""

Por las siguientes líneas:

INTERFACESv4="[NOMBRE_INTERFAZ_RED]"
#INTERFACESv6=""

INTERFACESv4: Se definen las interfaces de red IPv4 en la que funcionará el servidor.
INTERFACESv6: Si no se tiene IPv6 configurado, se debe comentar la línea.

Usa el siguiente comando para poder ver las interfaces del equipo en caso necesario:

ip a

1.3. Ahora es necesario editar el siguiente fichero para configurar el servicio DHCP:

nano /etc/dhcp/dhcpd.conf

1.3.1. Dentro de este fichero añadiremos la siguiente línea para poder obtener la arquitectura del cliente:

option arch code 93 = unsigned integer 16;

1.3.2. Además configuraremos el tiempo de concesión de las IPs:

default-lease-time [TIEMPO_SEGUNDOS];
max-lease-time [TIEMPO_SEGUNDOS];
min-lease-time [TIEMPO_SEGUNDOS];

default-lease-time: Tiempo base en segundos de la concesión de una configuración IP dada.
max-lease-time: Tiempo máximo en segundos de la concesión.
min-lease-time: Tiempo mínimo en segundos de la concesión.

1.3.3. También añadiremos esta configuración que permite crear una subnet donde se define la configuración IP a coneceder:

subnet [DIRECCIÓN_RED] netmask [MÁSCARA] {
    # Configuración básica
    option routers [VALOR];
    next-server [VALOR];

    # Configuración DNS
    option domain-name "[VALOR]";
    option domain-name-servers [VALOR];

    # Configuración de rangos
    range [IP_INICIAL] [IP_FINAL];
    range [IP_INICIAL_2] [IP_FINAL_2];
    # ...

    # Permitr arranque por red
    allow booting;
    allow bootp;

    # Obtener la arqitectura del cliente 
    # y seleccionar el fichero de arranque
    if option arch = 00:07 or option arch 00:09 {
        filename "bootnetx64.efi"; # UEFI
    } else {
        filename "pxelinux.0"; # BIOS
    }
}

Sintaxis básica:

subnet: La dirección de red IPv4.
netmask: Máscara punteada de la red.

Configuración básica:

routers: La dirección IPv4 del enrutador.
next-server: La dirección IPv4 del servidor TFTP donde se aloja el fichero de arranque.

Configuración DNS:

domain-name: El nombre del dominio de la red entre comillas.
domain-name-servers: La(s) dirección(es) IPv4 de lo(s) servidor(es) DNS separados por comas.

Configuración de rangos:

range: Se define la primera y última dirección IPv4 que se repartirán de ese rango. Gracias a los rangos se pueden excluir ciertas IPs reservadas.

El resto de líneas no deben de modificarse.


1.4. Por último habilitaremos y reiniciaremos el servicio DHCP para aplicar los cambios, además de verificar si tiene errores:

systemctl enable isc-dhcp-server
systemctl restart isc-dhcp-server
systemctl status isc-dhcp-server

2. Configuración del servidor TFTP

2.1. Instalaremos el servicio correspondiente en el servidor:

apt-get update
apt-get install tftpd-hpa

El servicio no requiere configuración pero si descargar los archivos necesarios.


2.2. Ahora, como usuario "root", nos situaremos en la ruta necesaria y descargaremos el fichero de arranque mediante CLI:

cd /srv/tftp
wget https://deb.debian.org/debian/dists/trixie/main/installer-amd64/current/images/netboot/netboot.tar.gz

2.2.1. Descomprimiremos el fichero tar.gz para obtener los archivos necesarios:

tar -xzf netboot.tar.gz

2.2.2. Y comprobaremos que estén los ficheros de arranque necesarios:

ls --color

2.3. Por último habilitaremos el servicio de tftpd-hpa:

systemctl enable tftpd-hpa

3. Comprobación del funcionamiento

En un cliente en el que se vaya a instalar Debian se debe entrar a la BIOS y configurar arranque por red IPv4.

En el caso de una máquina virtual de VirtualBox, se debe tener la MV cliente y servidor en una misma red NAT (sin DHCP activado) y, en la pestaña de "Sistema" de la configuración, se debe habilitar el arranque por red y subir su prioridad de arranque por encima de otros.

Una vez se inicie el sistema, normalmente requerirá al instante de obtener una configuración IP y un fichero de arranque que se pulse la tecla F12 u otra.

Una vez pulsado, debería aparecer el menú de instalación de Debian.

Para comprobar que el servidor esté concediendo direcciones IPs se puede usar el siguiente comando:

dhcp-lease-list
