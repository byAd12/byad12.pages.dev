Requisitos iniciales

El servidor debe tener una IPv4 estática y un hostname identificativo:

ip a
cat /etc/hostname

Pasos de configuración

Lograr una instalación desatendida


1. Configuración del servidor DHCP

1.1. Instalaremos el servicio correspondiente en el servidor:

apt-get update
apt-get install -y isc-dhcp-server

1.2. A continuación hay que modificar el siguiente archivo para configurar las interfaces de red:

nano /etc/default/isc-dhcp-server

Dentro de este archivo se deben reemplazar las siguientes líneas:

INTERFACESv4=""
INTERFACESv6=""

Por las siguientes líneas:

INTERFACESv4="[NOMBRE_INTERFAZ_RED]"
#INTERFACESv6=""
Configuración de tiempos de concesión 
INTERFACESv4=""
 Se definen las interfaces de red IPv4 en la que funcionará el servidor. 
INTERFACESv6=""
 Si IPv6 no está configurado se debe comentar esta línea.

Usa el siguiente comando para poder ver las interfaces del equipo en caso necesario:

ip a

1.3. Ahora es necesario editar el siguiente archivo para configurar el servicio DHCP:

nano /etc/dhcp/dhcpd.conf

1.3.1. Dentro de este archivo añadiremos la siguiente línea para poder obtener la arquitectura del cliente:

option arch code 93 = unsigned integer 16;

1.3.2. Además configuraremos el tiempo de concesión de las IPs:

default-lease-time [SEGUNDOS];
max-lease-time [SEGUNDOS];
min-lease-time [SEGUNDOS];
Configuración de tiempos de concesión 
default-lease-time [SEGUNDOS];
 Tiempo base en segundos de la concesión IP. 
max-lease-time [SEGUNDOS];
 Tiempo máximo en segundos de la concesión IP. 
min-lease-time [SEGUNDOS];
 Tiempo mínimo en segundos de la concesión IP.

1.3.3. La configuración de las concesiones IP se configuran con el siguiente bloque:

subnet [DIRECCIÓN_RED] netmask [MÁSCARA] {
    # Configuración básica
    option routers [VALOR];
    next-server [VALOR];

    # Configuración DNS
    option domain-name "[VALOR]";
    option domain-name-servers [VALOR];

    # Configuración de rangos
    range [IP_INICIAL] [IP_FINAL];
    range [IP_INICIAL_2] [IP_FINAL_2];
    # ...

    # Permitir arranque por red
    allow booting;
    allow bootp;

    # Obtener la arquitectura del cliente 
    # y seleccionar el archivo de arranque
    if option arch = 00:07 or option arch 00:09 {
        filename "bootnetx64.efi"; # UEFI
    } else {
        filename "pxelinux.0"; # BIOS
    }
}
Inicio de la configuración básica 
subnet [DIRECCIÓN_RED] netmask [MÁSCARA] {
 La dirección de red IPv4 y su máscara en formato punteado.
Configuración básica 
option routers [VALOR];
 La dirección IPv4 del enrutador. 
next-server [VALOR];
 La dirección IPv4 del servidor TFTP donde se aloja el archivo de arranque.
Configuración DNS 
option domain-name "[VALOR]";
 El nombre del dominio de la red entre comillas. 
option domain-name-servers [VALOR];
 La(s) dirección(es) IPv4 de lo(s) servidor(es) DNS separados por comas. Estos servidores deben poder resolver el mirror para instalar Debian.
Configuración de rangos 
range [IP_INICIAL] [IP_FINAL];
 Se define la primera y última dirección IPv4 que se repartirán de ese rango. Gracias a los rangos se pueden excluir ciertas IPs reservadas.

Las siguientes líneas no deben modificarse:

Habilitar servicios rangos 
allow booting;
 Permite que el servidor proporcione toda la información necesaria para el arranque por red (PXE). 
allow bootp;
 Habilita soporte para clientes que usan el protocolo BOOTP.
Selección del archivo de arranque
Gracias al condicional ip option ... se permite detectar la arquitectura del cliente para elegir el archivo correcto. 
filename "bootnetx64.efi";
 Este archivo se usa en sistemas UEFI. 
filename "pxelinux.0";
 Este archivo se usa en sistemas BIOS.
Si solo hay equipos sin UEFI en la red se puede quitar todo el bloque condicional y poner simplemente filename "pxelinux.0";.

1.4. Por último habilita y reinicia el servicio DHCP para aplicar los cambios, además de verificar si hay errores:

systemctl enable isc-dhcp-server
systemctl restart isc-dhcp-server
systemctl status isc-dhcp-server

2. Configuración del servidor TFTP

2.1. Instalaremos el servicio correspondiente en el servidor:

apt-get update
apt-get install tftpd-hpa

El servicio no requiere configuración pero si descargar los archivos necesarios.


2.2. Ahora, como usuario "root", nos situaremos en la ruta necesaria y descargaremos el archivo de arranque mediante CLI:

cd /srv/tftp
wget https://ftp.debian.org/debian/dists/trixie/main/installer-amd64/current/images/netboot/gtk/netboot.tar.gz

La URL corresponde al archivo comprimido en formato tar.gz, alojado en la página de netinst de Debian. Para obtenerla, se debe seleccionar la arquitectura del cliente en la sección "Arranque por red" y, a continuación, copiar la ruta del archivo netboot/netboot.tar.gz.

Importante: El repositorio donde se saque este archivo debe ser el mismo con el que se instalará el sistema (paso 4 - Gestión de paquetes).

2.2.1. Descomprimiremos el archivo tar.gz para obtener los archivos necesarios:

tar -xzf netboot.tar.gz

2.2.2. Y comprobaremos que estén los archivos de arranque necesarios:

ls --color

2.3. Por último habilitaremos el servicio de tftpd-hpa:

systemctl enable tftpd-hpa

3. Comprobación del funcionamiento

En un cliente en el que se vaya a instalar Debian se debe entrar a la BIOS y habilitar el arranque por red IPv4.

En el caso de una máquina virtual de VirtualBox, se debe tener la MV cliente y servidor en una misma red NAT (sin DHCP activado) y, en la pestaña de "Sistema" de la configuración, se debe habilitar el arranque por red y subir su prioridad de arranque por encima de otros.

Una vez se inicie el sistema, normalmente requerirá al instante de obtener una configuración IP y un archivo de arranque que se pulse la tecla F12 u otra.

Una vez pulsado, debería aparecer el menú de instalación de Debian.

Para comprobar que el servidor esté concediendo direcciones IPs se puede usar el siguiente comando:

dhcp-lease-list

4. Crear el archivo de respuestas

4.1. Crearemos un archivo con extensión .cfg dentro de /srv/tftp con las siguientes líneas de respuestas según las necesidades de la instalación. Puedes agrupar varios archivos de respuestas en una carpeta conjunta para una mejor organización. Un ejemplo completo de un archivo de respuestas es el preseed de Debian.

Ten en cuenta que cada línea que se añada en el archivo quitará una o varias preguntas del proceso de instalación de Debian:


Configuración regional:

País, idioma y codificación de caracteres (es_ES.UTF-8 - "español", "España", "UTF-8"):

d-i debian-installer/locale string <es_ES.UTF-8>

Distribución del teclado (es - "español"):

d-i keyboard-configuration/xkb-keymap select <es>

Las siguientes 3 opciones de configuración de red deben especificarse juntas:

Seleccionar la interfaz de red con conexión (auto - se detecta automáticamente):

d-i netcfg/choose_interface select auto

Definir el hostname del equipo [DHCP puede sobreescribirlo]:

d-i netcfg/get_hostname string <nombre_host>

Definir el dominio del equipo [DHCP puede sobreescribirlo]:

d-i netcfg/get_domain string <nombre_dominio>

En equipos con tarjetas de red Wi-Fi se puede deshabilitar WEP (Wired Equivalent Privacy), que está obsoleto y sustituido por WPA:

d-i netcfg/wireless_wep string

Gestión de usuarios:

Definir la contraseña del usuario root:

passwd passwd/root-password password contraseña_segura
passwd passwd/root-password-again password contraseña_segura

Crear un nuevo usuario local [se solicita obligatoriamente]:

d-i passwd/user-fullname string <nombre_completo_usuario>
d-i passwd/username string <nombre_usuario>

d-i passwd/user-password password <contraseña_segura>
d-i passwd/user-password-again password <contraseña_segura>

Asignar la contraseña al usuario local mediante un hash MD5 [opcional]:

d-i passwd/user-password-crypted password <Hash_MD5>

Asignar un UID al usuario local [opcional]:

d-i passwd/user-uid string <número_uid>

Gestión de la zona horaria:

Establecer el reloj en UTC:

d-i clock-setup/utc boolean true

Establecer la zona horaria actual (Europe/Madrid - se saca con la carpeta /usr/share/zoneinfo/):

d-i time/zone string <Europe/Madrid>

Establecer la hora correcta del reloj por NTP:

d-i clock-setup/ntp boolean true

Se puede elegir que servidor NTP se quiere usar [opcional]:

d-i clock-setup/ntp-server string <ntp.roa.es>

Particionar el disco:

Primero hay que establecer el método de particionado (regular, lvm, crypto):

d-i partman-auto/method string regular

Eliminar configuraciones LVM previas si existen [opcional]:

d-i partman-lvm/device_remove_lvm boolean true

Eliminar configuraciones RAID previas si existen [opcional]:

d-i partman-md/device_remove_md boolean true

Elegir el tipo de particionado:

  • atomic - Todos los archivos en una partición.
  • home - La partición /home se separa del resto.
  • multi - Las particiones /home, /var, /tmp se separan del resto.
d-i partman-auto/choose_recipe select <atomic>

Después de configurar el particionado se debe especificar que se aplique el proceso sin más preguntas:

d-i partman-partitioning/confirm_write_new_label boolean true
d-i partman/choose_partition select finish
d-i partman/confirm boolean true
d-i partman/confirm_nooverwrite boolean true

En la sección de "Archivo de respuestas: crear particiones manualmente" se explica detalladamente como crear cada partición necesaria gestionando manualmente su espacio en disco.


Gestión del mirror / repositorio:

Ya que netinst requiere el uso de red, hay que elegir que mirror se usará para la instalación del sistema:

d-i mirror/country string manual
d-i mirror/http/hostname string deb.debian.org
d-i mirror/http/directory string /debian
d-i mirror/http/proxy string

Configuraciones de personalización:

Para rechazar el envío opcional de estadísticas a los desarrolladores de Debian:

popularity-contest popularity-contest/participate boolean false

Instalar el sistema:

Para instalar el sistema estándar con interfaz gráfica (GNOME, MATE...) y un entorno de escritorio:

tasksel tasksel/first multiselect standard, desktop
tasksel tasksel/desktop multiselect <gnome / mate>

Para instalar el sistema estándar sin interfaz gráfica ni entorno de escritorio:

tasksel tasksel/first multiselect standard

Instalar el gestor de arranque GRUB:

Para instalar GRUB si solo está Debian instalado:

d-i grub-installer/only_debian boolean true

Para instalar GRUB si hay otro sistema operativo instalado a parte de Debian:

d-i grub-installer/with_other_os boolean true

Para instalar GRUB en el sector MBR del disco:

d-i grub-installer/bootdev string /dev/sda

Configuración final:

Para que no se abra un diálogo de información cuando la instalación acabe y que además reinicie el sistema:

d-i finish-install/reboot_in_progress note

Todas las líneas de esta sección crea una instalación completamente desatendida. Hay configuraciones adicionales que se explican al final de la guía.

5. Modificar el menú de arranque en la instalación

5.1. Primero tendremos que editar el siguiente archivo:

nano /srv/tftp/debian-installer/amd64/boot-screens/txt.cfg

Dentro de ese archivo nos encontraremos bloques de configuración como el siguiente:

label install
        menu label ^Install
        kernel debian-installer/amd64/linux
        append vga=788 initrd=debian-installer/amd64/initrd.gz --- quiet
Formato del bloque de configuración 
label <...>
 Nombre identificativo para la entrada. 
menu label <...>
 Es la descripción de la entrada que el usuario verá. El símbolo ^ indica que la letra que tenga a continuación será el acceso rápido y, cuando esta letra se pulse, se seleccionará la opción (este símbolo es opcional). 
kernel <...>
 Es el kernel a cargar. 
append
 Define las opciones de la instalación. 
vga=788
 Define la resolución de la pantalla a usar para la instalación. 
initrd=...
 Define la ruta al archivo de Initial RAM Disk, que contiene controladores y herramientas necesarias para montar el sistema. 
---
 Define que todo lo que tenga a la derecha se procese por el kernel. 
quiet
 Indica que no muestre los logs durante la instalación.

5.2. Lo que hay que hacer es añadir debajo del bloque de configuración existente otro bloque separado por una línea vacía:

label instalacion-desatendida-gnome
        menu label Instalación desatendida de ^GNOME
        kernel debian-installer/amd64/linux
        append vga=788 initrd=debian-installer/amd64/initrd.gz auto=true priority=critical preseed/url=tftp://<IP_Servidor_TFTP>/<Archivo_Respuestas.cfg> --- quiet
Opciones de configuración nuevas 
auto=true
 Indica que todos los pasos se deben procesar cuando se cargue el archivo de respuestas. 
priority=critical
 Especifica que no deben aparecer diálogos si no son de prioridad crítica. 
preseed/url=tftp://<IP_Servidor_TFTP>/<Archivo_Respuestas.cfg>
 Indica la ruta al servidor TFTP donde se encuentra el archivo de respuestas.

Si no se usa priority=critical, el instalador mostrará el diálogo para configurar el hostname, aunque ya esté definido en el archivo de respuestas. Esto evita que varios equipos se instalen con el mismo nombre en la misma red.

Si se usa, el diálogo se omite y todos los equipos usarán el mismo hostname definido en el archivo, por lo que después será necesario cambiarlo manualmente en cada equipo.


5.3. Ahora guarda el archivo y ya estaría funcionando la nueva configuración.


Nota: Se pueden añadir varias entradas en el menú de arranque separados por una línea en blanco, cada una con un archivo de respuestas distinto. Esto permite instalar diferentes sistemas con configuraciones específicas. Para identificarlos fácilmente, cada entrada debe tener un label diferente.

Configuración opcional

Archivo de respuestas


Archivo de respuestas: instalar paquetes adicionales

Se puede instalar paquetes específicos separado por espacios, como por ejemplo "wireshark nfs-kernel-server":

d-i pkgsel/include string <Paquete1> <Paquete2> <...>

Archivo de respuestas: ejecución de comandos adicionales

Con el siguiente comando se especifica qué comandos se ejecutarán al finalizar la instalación:

d-i preseed/late_command string \
    in-target <Comando 1>; \
    in-target <Comando 2>; \
    ...;
  • in-target - Indica que el comando se va a ejecutar en el sistema a instalar y no en el instalador.
  • El ; indica el final de cada línea de comando a ejecutar y el \ indica un salto de línea de la configuración de preseed.

Para definir una ruta del instalador se puede hacer de 2 formas que dan el mismo resultado:

in-target echo /<Ruta_Absoluta>
 Ejecuta el comando directamente en el sistema a instalar. 
echo /target/<Ruta_Absoluta>
 Ejecuta el comando en el instalador pero modificando los archivos del sistema a instalar.

Una manera útil de comprobar que realmente se está utilizando "/target" es ejecutar el instalador y, al pulsar una de las teclas F1 a F5, se abrirá una terminal. Desde esa terminal se puede ejecutar el siguiente comando, con el que se observará que las particiones se montan en "/target": 

mount

Archivo de respuestas: compartir una carpeta por NFS

1. Para compartir una carpeta por NFS primero se requiere instalar el servicio "nfs-kernel-server":

d-i pkgsel/include string nfs-kernel-server

Suele ser necesario poner esta línea antes de la configuración del paso 2.


2. Luego hay que especificar que se debe ejecutar una serie de comandos al finalizar la instalación:

d-i preseed/late_command string \
    mkdir -p /target/<Ruta_Absoluta>; \
    echo "/<Ruta_Absoluta>    *(ro)" >> /target/etc/exports; \
    in-target systemctl enable nfs-kernel-server;
  • mkdir - Se debe crear la carpeta antes de compartirla, especificando su ruta local absoluta.
  • echo - Ahora se añade una línea en /etc/exports, donde se especifica la ruta de la carpeta a compartir y sus permisos (ro, rw).
  • systemctl - Se indica que se debe habilitar el servicio de NFS para que arranque cuando se inicie el sistema.

En la sección de "ejecución de comandos adicionales" se puede ver la explicación sobre "Target".

Archivo de respuestas: montar una carpeta por NFS

1. Para compartir una carpeta por NFS primero se requiere instalar el servicio "nfs-common":

d-i pkgsel/include string nfs-common

Suele ser necesario poner esta línea antes de la configuración del paso 2.


2. Luego hay que especificar que se debe ejecutar una serie de comandos al finalizar la instalación:

d-i preseed/late_command string \
    mkdir -p /target/<Carpeta_Local>; \
    echo "<IP_Servidor>:/<Carpeta_Remota>    /<Carpeta_Local>    nfs    defaults,_netdev,nfsvers=3    0    0" >> /target/etc/fstab;
  • mkdir - Se debe crear la carpeta antes de montarla, especificando la ruta local absoluta.
  • echo - Ahora se añade una línea en /etc/fstab, donde se especifica la carpeta NFS a montar en el sistema local cada vez que el sistema se inicia.
  • Hay que usar el tabulador para separar los elementos del comando "echo".

En la sección de "ejecución de comandos adicionales" se puede ver la explicación sobre "Target".


Verificación - Una vez instalado el sistema se puede comprobar con el siguiente comando si se ha montado correctamente:

mount | grep <Carpeta_Local>

Archivo de respuestas: crear particiones manualmente

A continuación se muestra la sintaxis básica para definir varias particiones en una misma receta:

d-i partman-auto/expert_recipe string \
    <nombre receta> ::                \
        <partición 1>                 \
            <Opciones>                \
        .                             \
        <partición 2>                 \
            <Opciones>                \
        .                             \
        <partición 3>                 \
            <Opciones>                \
        .

El símbolo \ sirve para continuar la línea en otra.


Formato de cada partición:

<Tamaño mínimo MB> <Prioridad MB> <Tamaño máximo MB> <Sistema de archivos> \
    <Opciones> \

La suma de los tamaños mínimos indica el espacio mínimo que debe tener el disco. Si el disco es más pequeño, la instalación fallará.

El espacio restante se reparte entre las particiones según su prioridad.

  • La prioridad se calcula restando la Prioridad con el Tamaño mínimo.
  • Este valor determina qué porcentaje del espacio libre recibe cada partición. La suma de todos los porcentajes debería ser 100.
    • Si una partición supera su tamaño máximo, solo usará el máximo permitido.
    • Para que una partición use todo el espacio restante, se puede indicar -1 como tamaño máximo.
    • Si ninguna partición tiene -1 y queda espacio sin asignar, la última partición ocupará el espacio restante.

Formato de las opciones de cada partición:

<Nombre Opción 1>{ <Atributos de la opción> } <Nombre Opción 2>{ <Atributos de la opción> } ...
Tipos de opciones de cada partición 
$primary{ }
 Indica que la partición debe ser primaria. No necesita atributos. 
$bootable{  }
 Indica que la partición es arrancable. No necesita atributos. 
method{ format }
 Indica que la partición debe ser formateada. 
use_filesystem{ } filesystem{ <Sistema de archivos> }
 Define qué sistema de archivos usar, como "ext4". 
mountpoint{ <Donde> }
 Define dónde se montará la partición en el árbol de directorios, como "/".

Ejemplo - Creación de una partición /boot, swap y otra con el resto de espacio:

d-i partman-auto/expert_recipe string                         \
      boot-root ::                                            \
              40 50 100 ext3                                  \
                      $primary{ } $bootable{ }                \
                      method{ format } format{ }              \
                      use_filesystem{ } filesystem{ ext3 }    \
                      mountpoint{ /boot }                     \
              .                                               \
              500 10000 1000000000 ext3                       \
                      method{ format } format{ }              \
                      use_filesystem{ } filesystem{ ext3 }    \
                      mountpoint{ / }                         \
              .                                               \
              64 512 300% linux-swap                          \
                      method{ swap } format{ }                \
              .

Una vez se configuren las particiones se debe indicar otras configuraciones avanzadas:

d-i partman-partitioning/confirm_write_new_label boolean true
d-i partman/choose_partition select finish
d-i partman/confirm boolean true
d-i partman/confirm_nooverwrite boolean true
