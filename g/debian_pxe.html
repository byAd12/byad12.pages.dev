Debian PXE

Automatizar la instalación de Debian

none

1. Configuración del servidor DHCP

1.1. Instalaremos el servicio correspondiente en el servidor:

apt-get update
apt-get install -y isc-dhcp-server

1.2. A continuación deberemos modificar el siguiente fichero para configurar las interfaces de red:

nano /etc/default/isc-dhcp-server

Dentro de este fichero reemplazaremos las siguientes líneas:

INTERFACESv4=""
INTERFACESv6=""

Por las siguientes líneas:

INTERFACESv4="[NOMBRE_INTERFAZ_RED]"
#INTERFACESv6=""

INTERFACESv4: Se definen las interfaces de red IPv4 en la que funcionará el servidor.
INTERFACESv6: Si no se tiene IPv6 configurado, se debe comentar la línea.

Usa el siguiente comando para poder ver las interfaces del equipo en caso necesario:

ip a

1.3. Ahora es necesario editar el siguiente fichero para configurar el servicio DHCP:

nano /etc/dhcp/dhcpd.conf

1.3.1. Dentro de este fichero añadiremos la siguiente línea para poder obtener la arquitectura del cliente:

option arch code 93 = unsigned integer 16;

1.3.2. Además configuraremos el tiempo de concesión de las IPs:

default-lease-time [TIEMPO_SEGUNDOS];
max-lease-time [TIEMPO_SEGUNDOS];
min-lease-time [TIEMPO_SEGUNDOS];

default-lease-time: Tiempo base en segundos de la concesión de una configuración IP dada.
max-lease-time: Tiempo máximo en segundos de la concesión.
min-lease-time: Tiempo mínimo en segundos de la concesión.

1.3.3. También añadiremos esta configuración que permite crear una subnet donde se define la configuración IP a coneceder:

subnet [DIRECCIÓN_RED] netmask [MÁSCARA] {
    # Configuración básica
    option routers [VALOR];
    next-server [VALOR];

    # Configuración DNS
    option domain-name "[VALOR]";
    option domain-name-servers [VALOR];

    # Configuración de rangos
    range [IP_INICIAL] [IP_FINAL];
    range [IP_INICIAL_2] [IP_FINAL_2];
    # ...

    # Permitr arranque por red
    allow booting;
    allow bootp;

    # Obtener la arqitectura del cliente 
    # y seleccionar el fichero de arranque
    if option arch = 00:07 or option arch 00:09 {
        filename "bootnetx64.efi"; # UEFI
    } else {
        filename "pxelinux.0"; # BIOS
    }
}

Sintaxis básica:

subnet: La dirección de red IPv4.
netmask: Máscara punteada de la red.

Configuración básica:

routers: La dirección IPv4 del enrutador.
next-server: La dirección IPv4 del servidor TFTP donde se aloja el fichero de arranque.

Configuración DNS:

domain-name: El nombre del dominio de la red entre comillas.
domain-name-servers: La(s) dirección(es) IPv4 de lo(s) servidor(es) DNS separados por comas.

Configuración de rangos:

range: Se define la primera y última dirección IPv4 que se repartirán de ese rango. Gracias a los rangos se pueden excluir ciertas IPs reservadas.

El resto de líneas no deben de modificarse.


1.4. Por último habilitaremos y reiniciaremos el servicio DHCP para aplicar los cambios, además de verificar si tiene errores:

systemctl enable isc-dhcp-server
systemctl restart isc-dhcp-server
systemctl status isc-dhcp-server

2. Configuración del servidor TFTP

2.1. Instalaremos el servicio correspondiente en el servidor:

apt-get update
apt-get install tftpd-hpa

El servicio no requiere configuración pero si descargar los archivos necesarios.


2.2. Ahora, como usuario "root", nos situaremos en la ruta necesaria y descargaremos el fichero de arranque mediante CLI:

cd /srv/tftp
wget https://deb.debian.org/debian/dists/trixie/main/installer-amd64/current/images/netboot/netboot.tar.gz

2.2.1. Descomprimiremos el fichero tar.gz para obtener los archivos necesarios:

tar -xzf netboot.tar.gz

2.2.2. Y comprobaremos que estén los ficheros de arranque necesarios:

ls --color

2.3. Por último habilitaremos el servicio de tftpd-hpa:

systemctl enable tftpd-hpa

3. Comprobación del funcionamiento

En un cliente en el que se vaya a instalar Debian se debe entrar a la BIOS y configurar arranque por red IPv4.

En el caso de una máquina virtual de VirtualBox, se debe tener la MV cliente y servidor en una misma red NAT (sin DHCP activado) y, en la pestaña de "Sistema" de la configuración, se debe habilitar el arranque por red y subir su prioridad de arranque por encima de otros.

Una vez se inicie el sistema, normalmente requerirá al instante de obtener una configuración IP y un fichero de arranque que se pulse la tecla F12 u otra.

Una vez pulsado, debería aparecer el menú de instalación de Debian.

Para comprobar que el servidor esté concediendo direcciones IPs se puede usar el siguiente comando:

dhcp-lease-list

4. Automatizar las preguntas de la instalación

4.1. Crearemos un fichero con extensión .cfg dentro de /srv/tftp con las siguientes líneas de respuestas según las necesidades de la instalación.

Ten en cuenta que cada línea que se añada en el fichero quitará una o varias preguntas del proceso de instalación de Debian:


Configuración regional:

País, idioma y codificación de carácteres (es_ES.UTF-8 - "español", "España", "UTF-8"):

d-i debian-installer/locale string <es_ES.UTF-8>

Distribución del teclado (es - "español"):

d-i keyboard-configuration/xkb-keymap select <es>

Las siguientes 3 opciones de configuración de red deben especificarse juntas:

Seleccionar la interfaz de red con conexión (auto - se detecta automáticamente):

d-i netcfg/choose_interface select auto

Definir el hostname del equipo [DHCP puede sobreescribirlo]:

d-i netcfg/get_hostname string <nombre_host>

Definir el dominio del equipo [DHCP puede sobreescribirlo]:

d-i netcfg/get_domain string <nombre_dominio>

En equipos con tarjetas de red Wi-Fi se puede deshabilitar WEP (Wired Equivalent Privacy), que está obsoleto y sustituido por WPA:

d-i netcfg/wireless_wep string

Gestión de usuarios:

Definir la contraseña del usuario root:

passwd passwd/root-password password contraseña_segura
passwd passwd/root-password-again password contraseña_segura

Crear un nuevo usuario local [se solicita obligatoriamente]:

d-i passwd/user-fullname string <nombre_completo_usuario>
d-i passwd/username string <nombre_usuario>

d-i passwd/user-password password <contraseña_segura>
d-i passwd/user-password-again password <contraseña_segura>

Asignar la contraseña al usuario local mediante un hash MD5 [opcional]:

d-i passwd/user-password-crypted password <Hash_MD5>

Asignar un UID al usuario local [opcional]:

d-i passwd/user-uid string <número_uid>

Gestión de la zona horaria:

Establecer el reloj en UTC:

d-i clock-setup/utc boolean true

Establecer la zona horaria actual (Europe/Madrid - se saca con la carpeta /usr/share/zoneinfo/):

d-i time/zone string <Europe/Madrid>

Establecer la hora correcta del reloj por NTP:

d-i clock-setup/ntp boolean true

Se puede elegir que servidor NTP se quiere usar [opcional]:

d-i clock-setup/ntp-server string <ntp.roa.es>

Particionar el disco:

Primero hay que establecer el método de particionado (regular, lvm, crypto):

d-i partman-auto/method string regular

Eliminar configuraciones LVM previas si existen [opcional]:

d-i partman-lvm/device_remove_lvm boolean true

Eliminar configuraciones RAID previas si existen [opcional]:

d-i partman-md/device_remove_md boolean true

Elegir el tipo de particionado:

  • atomic - Todos los archivos en una partición
  • home - La partición /home se separa del resto
  • multi - Las particiones /home, /var, /tmp se separan del resto
d-i partman-auto/choose_recipe select <atomic>

Después de configurar el particionado se debe especificar que se aplique el proceso sin más preguntas:

d-i partman-partitioning/confirm_write_new_label boolean true
d-i partman/choose_partition select finish
d-i partman/confirm boolean true
d-i partman/confirm_nooverwrite boolean true

Gestión de paquetes:

Hay que elegir que mirror se usará para la instalación del sistema, ya que netinst requiere el uso de red:

d-i mirror/country string manual
d-i mirror/http/hostname string <ftp.es.debian.org>
d-i mirror/http/directory string /debian
d-i mirror/http/proxy string

Configuraciones de personalización:

Para rechazar el envío opcional de estadísticas a los desarrolladores de Debian:

popularity-contest popularity-contest/participate boolean false

Selección de paquetes a instalar:

Para instalar un entorno gráfico de escritorio con GNOME:

tasksel tasksel/first multiselect standard, desktop
tasksel tasksel/desktop multiselect <gnome>

Instalar el gestor de arranque GRUB:

Para instalar GRUB si solo está Debian instalado:

d-i grub-installer/only_debian boolean true

Para instalar GRUB si hay otro sistema operativo instalado a parte de Debian:

d-i grub-installer/with_other_os boolean true

Para instalar GRUB en el sector MBR del disco:

d-i grub-installer/bootdev string /dev/sda

Configuración final:

Para que no se abra un diálogo de información cuando la instalación acabe y que además reinicie el sistema:

d-i finish-install/reboot_in_progress note

5. Modificar el menú de arranque en la instalación

5.1. Primero tendremos que editar el siguiente fichero:

nano /srv/tftp/debian-installer/amd64/boot-screens/txt.cfg

Dentro de ese fichero nos encontraremos bloques de configuración como el siguiente:

label install
        menu label ^Install
        kernel debian-installer/amd64/linux
        append vga=788 initrd=debian-installer/amd64/initrd.gz --- quiet
Formato del bloque de configuración 
label <...>
 Nombre identificativo para la entrada. 
menu label <...>
 Es la descripción de la entrada que el usuario verá. El símbolo ^ indica que la letra que tenga a continuación será el acceso rápido y, cuando esta letra se pulse, la opción se activará (este símbolo es opcional). 
kernel <...>
 Es el kernel a cargar. 
append
 Define las opciones de la instalación. 
vga=788
 Define la resolución de la pantalla a usar para la instalacion. 
initrd=...
 Define la ruta al archivo de Initial RAM Disk, que contiene controladores y herramientas necesarias para montar el sistema. 
---
 Define que todo lo que tenga a la derecha se procese por el kernel. 
quiet
 Indica que no muestre los logs durante la instalación.

5.2. Lo que hay que hacer es añadir debajo del bloque de configuración existente otro bloque separado por una línea vacía:

label instalacion-desatendida-gnome
        menu label Instalación desatendida de ^GNOME
        kernel debian-installer/amd64/linux
        append vga=788 initrd=debian-installer/amd64/initrd.gz auto=true priority=critical pressed/url=tftp://<IP_Servidor_TFTP>/<Archivo_Respuestas.cfg> --- quiet
Opciones de configuración nuevas 
auto=true
 Indica que todos los pasos se deben procesar cuando se cargue el archivo de respuestas. 
priority=critical
 Especifica que no deben aparecer diálogos si no son de prioridad crítica. 
pressed/url=tftp://<IP_Servidor_TFTP>/<Archivo_Respuestas.cfg>
 Indica la ruta al servidor TFTP donde se encuentra el archivo de respuestas.

5.3. Ahora simplemente se guarda el fichero y ya estaría funcionando la nueva configuración.
