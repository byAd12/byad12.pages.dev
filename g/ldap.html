LDAP - Servidor

Servicio de Directorio

none

Instalación

Primero instalaremos el servidor LDAP (slapd) y las utilidades (ldap-utils):

apt-get update
apt install slapd ldap-utils -y

Al instalarlo nos pedirá una contraseña para la gestión del servicio del servidor (slapd).

Comprobación

El servicio debe estar en modo active:

systemctl status slapd

El servicio slapd debe tener un puerto en modo LISTEN:

ss -pltu | grep slapd

Al no haber configurado nada, deberá devolver NODOMAIN:

slapcat

La conexión al servicio la haremos con el usuario anonymous si no definimos nada:

ldapwhoami -x

Formato de archivo

Los archivos .ldif sirven para crear varios elementos a la vez para luego añadirlos a la BB.DD. con el comando ldapadd:

# Elemento 1
dn: <URI>
    <atributos>: <valor>
    ...

[Salto de línea obligatorio antes de añadir otro elemento]
# Elemento 2
dn: <URI>
    <atributos>: <valor>
    ...

dn (Distingushed Name): Diferencia a todos los objetos de la BB.DD.

Configuración del Servicio

Una vez instalemos los servicios lo configuraremos con el siguiente comando:

dpkg-reconfigure slapd

Respuestas que va a pedir: No; [DOMINIO]; [NOMBRE_EMPRESA]; Meter la contraseña; No; Si.

Podemos comprobar el estado del servidor:

slapcat

Ahora comprobaremos que podemos entrar al servicio con el usuario admin:

ldapwhoami -x -D "cn=admin,dc=[NOMBRE_DOMINIO],dc=[TLD_DOMINIO_SIN_PUNTO]" -W

Luego ponemos la contraseña y si devuelve lo que escribimos está bien configurado.

Crear una Unidad Organizativa

1. Hay que crear un archivo .ldif y abrirlo.

2. En su interior pondremos el siguiente contenido:

dn: ou=[UO],dc=[NOMBRE_DOMINIO],dc=[TLD_DOMINIO_SIN_PUNTO]
objectClass: top
objectClass: organizationalUnit
ou: [UO]

UO: Nombre personalizado que tendrá la unidad organizativa.
NOMBRE_DOMINIO: Nombre del dominio sin el (.es, .com ..).
TLD_DOMINIO: TLD del dominio (es, com ..) sin el punto.

3. Añadiremos el archivo a la base de datos:

ldapadd -x -D cn=admin,dc=[NOMBRE_DOMINIO],dc=[TLD_DOMINIO_SIN_PUNTO] -W -f [ARCHIVO].ldif

ARCHIVO: Nombre del archivo creado en el paso 1.

4. Hay que comprobar si en la base de datos está la nueva unidad organizativa:

slapcat

Crear un Grupo

1. Hay que crear un archivo .ldif y abrirlo.

2. En su interior pondremos el siguiente contenido:

dn: cn=[NOMBRE_GRUPO],ou=[UO],dc=[NOMBRE_DOMINIO],dc=[TLD_DOMINIO_SIN_PUNTO]
objectClass: top
objectClass: posixGroup
gidNumber: [N_GID]
cn: [NOMBRE_GRUPO]

NOMBRE_GRUPO: Nombre personalizado que tendrá el grupo.
UO: Unidad organizativa a la que pertenecerá.
NOMBRE_DOMINIO: Nombre del dominio sin el (.es, .com ..).
TLD_DOMINIO: TLD del dominio (es, com ..) sin el punto.

N_GID: Número entero identificador del grupo.

3. Añadiremos el archivo a la base de datos:

ldapadd -x -D cn=admin,dc=[NOMBRE_DOMINIO],dc=[TLD_DOMINIO_SIN_PUNTO] -W -f [ARCHIVO].ldif

4. Hay que comprobar si en la base de datos está el nuevo grupo:

slapcat

Crear un Usuario

1. Crearemos el hash de la contraseña con el siguiente comando:

slappasswd

Solo hay que ingresar la contraseña y nos devolverá un hash como por ejemplo: {SSHA}jJ/Sl2m---5HeZJP.

2. Hay que crear un archivo .ldif y abrirlo.

3. En su interior pondremos el siguiente contenido:

dn: uid=[NOMBRE_USUARIO],ou=[UO],dc=[NOMBRE_DOMINIO],dc=[TLD_DOMINIO_SIN_PUNTO]
objectClass: top
objectClass: account
objectClass: posixAccount
uid: [NOMBRE_USUARIO]
cn: [NOMBRE_COMÚN]
userPassword: [CONTRASEÑA]
gecos: [NOMBRE_COMPLETO]
uidNumber: [N_UID]
gidNumber: [N_GID]
homeDirectory: /home/[NOMBRE_USUARIO]
loginShell: [SHELL]

UO: Unidad organizativa a la que pertenecerá.
NOMBRE_DOMINIO: Nombre del dominio sin el (.es, .com ..).
TLD_DOMINIO: TLD del dominio (es, com ..) sin el punto.

NOMBRE_USUARIO: Nombre personalizado que tendrá el usuario.
NOMBRE_COMÚN: Nombre común personalizado que tendrá el usuario.
NOMBRE_COMPLETO: Nombre completo personalizado que tendrá el usuario.
CONTRASEÑA: La contraseña en hash del usuario - el resultado del paso 1.

N_UID: Número entero identificador del usuario. Los siguientes usuarios creados, recibirán un valor +1 alto que este.
N_GID: Número entero identificador del grupo. 0 = Ninguno.

SHELL: /bin/bash, /bin/sh.

4. Añadiremos el archivo a la base de datos:

ldapadd -x -D cn=admin,dc=[NOMBRE_DOMINIO],dc=[TLD_DOMINIO_SIN_PUNTO] -W -f [ARCHIVO].ldif

5. Hay que comprobar si en la base de datos está el nuevo usuario:

slapcat

Crear un Usuario - Perfil Móvil

Hay que hacer lo mismo que en el apartado "Crear un Usuario" pero cambiando la ruta de homeDirectory a la carpeta donde se almacenarán estos perfiles:

homeDirectory: /[CARPETA_MÓVIL]/[NOMBRE_USUARIO]

Modificar un objeto

1. Crear un archivo .ldif:

2. Poner en su interior:

dn: [DN_COMPLETO]
changetype: modify
replace: [PROPIEDAD_CAMBIAR]
[PROPIEDAD_CAMBIAR]: [VALOR_CAMBIAR]

Nota: Para hacer 2 cambios se pone uno debajo del otro separado por una línea en blanco.

DN_COMPLETO: Identificador del objeto dentro de la base de datos.

PROPIEDAD_CAMBIAR: Nombre de la propiedad. Por ejemplo el "uid", "gid", etc.
VALOR_CAMBIAR: El valor que se le va a dar a esa propiedad.

3. Modificar el registro de la base de datos:

ldapmodify -x -D "cn=admin,dc=[NOMBRE_DOMINIO],dc=[TLD_DOMINIO_SIN_PUNTO]" -W -f [ARCHIVO].ldif

4. Hay que comprobar si se hizo el cambio en la base de datos:

slapcat

Eliminar un objeto

Hay 2 formas:

1. Con el comando ldapdelete:

ldapdelete -x -D "cn=admin,dc=[NOMBRE_DOMINIO],dc=[TLD_DOMINIO_SIN_PUNTO]" -W "[DN_COMPLETO]"

DN_COMPLETO: Identificador del objeto dentro de la base de datos.

2. Creando un archivo .ldif y agregando dentro el siguiente contenido:

dn: [DN_COMPLETO]
changetype: delete
ldapmodify -x -D "cn=admin,dc=[NOMBRE_DOMINIO],dc=[TLD_DOMINIO_SIN_PUNTO]" -W -f [ARCHIVO].ldif

Buscar un objeto

1. Con el comando ldapsearch podemos buscar objetos identificados por su URI:

ldapsearch -xLLL -b "dc=[NOMBRE_DOMINIO],dc=[TLD_DOMINIO_SIN_PUNTO]" [BÚSQUEDA] [ENSEÑAR]

BÚSQUEDA: El objeto que vamos a buscar según un valor. Por ejemplo: "uid=xx", "gid=xx".

ENSEÑAR: Es opcional. Son los valores del objeto a buscar que se quieren enseñar. Por ejemplo: "cn", "uid", "gid".

LDAP - Cliente

Conexión de un cliente al servicio LDAP de un servidor

1. Primero añadiremos un registro DNS interno que apunte al dominio LDAP mediante el siguiente fichero:

nano /etc/hosts

Añadiremos una línea en el fichero con el siguiente formato:

[IP_SERVIDOR]    [NOMBRE_DOMINIO].[TLD_DOMINIO_SIN_PUNTO]

2. Primero instalaremos los servicios necesarios:

apt install -y libpam-ldap libnss-ldap nss-updatedb libnss-db nfs-common nslcd

Va a pedir una serie de preguntas, a las cuales tendremos que responder de la siguiente manera:

1. Cual es la URL del servidor - ldap://[IP_SERVIDOR]

2. Cual es el dominio de LDAP - dc=[NOMBRE_DOMINIO],dc=[TLD_DOMINIO_SIN_PUNTO]

3. Seleccionar "Version3"

4. Seleccionar "Sí"

5. Seleccionar "No"

6. Cual es la URI del usuario administrador - cn=admin,dc=[NOMBRE_DOMINIO],dc=[TLD_DOMINIO_SIN_PUNTO]

7. Va a pedir la contraseña del usuario administrador por protección antes de unirte al dominio

8. Seleccionar "msd"

Si el proceso de instalación falla o no pregunta el paso 8, hay que reconfigurarlo:

dpkg-reconfigure ldap-auth-config

3. Una vez configurado hay que editar el siguiente fichero para definir que se pueda iniciar sesión con usuarios de LDAP:

nano /etc/nsswitch.conf

De las 3 primeras líneas reemplazaremos la palabra compact a files ldap.

4. Ahora configuraremos PAM. Primero haremos lo siguiente:

update /upgrade

Y a continuación habilitaremos todo:

pam-auth-update

5. Comprobaremos que podemos ver los usuarios del servidor en el cliente:

getent passwd

6. Del siguiente fichero tendremos que quitar de una línea la palabra use_authtok:

nano /etc/pam.d/common-password

7. Reiniciaremos el equipo para que se apliquen los cambios:

reboot -r now

8. Reiniciaremos el servicio de LDAP:

service libnss-ldap restart

9. Y por último intentaremos iniciar sesión con un usuario del servidor en el equipo cliente:

login [USUARIO_SERVIDOR]
