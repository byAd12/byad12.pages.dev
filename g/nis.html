NIS

Network Information Service

Instalación

Tanto en el cliente como en el servidor tendremos que instalar el servicio NIS:

apt update
apt install -y nis

Configuración en el servidor

1. Primero editaremos el siguiente archivo para poder definir un nombre de dominio como por ejemplo iso.asir:

nano /etc/defaultdomain

Ahora, el dominio que hayamos configurado debe enseñarse con el comando:

nisdomainname

Además le asignaremos a este servidor su dominio DNS correspondiente:

hostnamectl set-hostname <Hostname_Servidor.dominio.tld>

2. A continuación editaremos la configuración de NIS:

nano /etc/default/nis

Dentro de este archivo definiremos la función del servidor con las siguientes líneas:

NISSERVER=master
NISCLIENT=false

Si este servidor va a actuar como esclavo, debe reemplazarse "master" por "slave".

3. Por seguridad definiremos la subred de IPs que podrán entrar a interactuar con el servidor:

nano /etc/ypserv.securenets

Ya que por defecto se permite el acceso a todas las subredes comentaremos las siguientes líneas con un # al principio de cada línea:

# 0.0.0.0    0.0.0.0
# ::/0

Y añadiremos una línea nueva donde definiremos la dirección de red y máscara, como por ejemplo:

255.255.255.0    192.168.1.0

4. Ahora estableceremos un registro DNS interno por cada cliente y servidor del dominio mediante el siguiente archivo:

nano /etc/hosts

Definiremos con nuevas líneas lo siguiente, con una separación entre elementos con tabuladores:

<IP_Servidor>    <Hostname_Servidor.dominio.tld>    <Hostname_Servidor>
<IP_Cliente>     <Hostname_Cliente.dominio.tld>     <Hostname_Cliente>
...

Este paso no es obligatorio si ya hay un servicio DNS real ejecutándose para resolver los nombres de dominio.

5. Si el servidor va a actuar como esclavo y además como cliente hay que modificar el siguiente archivo:

nano /etc/yp.conf

Hay que añadir una nueva línea al final del archivo para definir cuál va a ser el servidor maestro:

ypserver <Hostname_Servidor_Master.dominio.tld>

6. Ahora generaremos la base de datos de todos los archivos del servidor como passwd, group, shadow, etc. para que los clientes lo añadan a su sistema:

Si el servidor actuará como master:

/usr/lib/yp/ypinit -m

Por defecto añadirá el hostname del servidor y, si no hay más servidores que definir, cerraremos el programa pulsando CTRL + D. Luego pedirá una confirmación en la que simplemente pondremos y.

Si el servidor actuará como esclavo / slave:

/usr/lib/yp/ypinit -s <Hostname_Servidor_Master.dominio.tld>

7. Ahora reiniciaremos todos los servicios y si hay fallos habrá que resolverlos:

systemctl restart rpcbind ypserv ypbind yppasswdd ypxfrd

8. Una vez que el paso anterior funcione sin ningún error, habrá que habilitar los servicios para que estén iniciados aunque se reinicie el equipo:

systemctl enable rpcbind ypserv ypbind yppasswdd ypxfrd

Actualización de la base de datos

En caso de que haya creado un nuevo usuario o grupo en el servidor maestro, habrá que aplicar estos cambios en la base de datos para que otros clientes puedan añadirlo a sus sistemas y que el resto de servidores esclavos repliquen la configuración:

make -C /var/yp

Si no hay cambios no actualizará nada, además el comando indica si un fichero fué modificado desde la última vez que se modificó la base de datos.

Problema - El cliente no tiene carpeta /home

La carpeta home del usuario no existe en el cliente ya que es una carpeta local del servidor, por ello no dejará iniciar sesión ya que el entorno gráfico necesita una carpeta personal para funcionar en la mayoría de los casos.

La mejor solución a ello es compartir la carpeta /home del servidor mediante NFS y luego montarla en los clientes, permitiendo así tener perfiles móviles.

Hay otra opción de la cual no se comparten las carpetas pero se define que cualquier usuario sin carpeta personal creada se creará automáticamente de manera local. Esto se usa cuando se quiere que los datos de un usuario esté en un equipo pero no en otro equipo.

Para configurarlo hay que editar el siguiente fichero:

nano /etc/pam.d/common-session

Y añadir la siguiente línea al final del fichero:

session	required	pam_mkhomedir.so	skel=/etc/skel/	umask=0022

Solución - Compartir la carpeta /home por NFS

Los siguientes pasos se explican mejor en la guía detallada sobre NFS.

1. Primero instalaremos el servicio de NFS:

apt install -y nfs-kernel-server

2. Ahora editaremos el fichero de configuración de NFS:

nano /etc/exports

Dentro de este fichero añadiremos la siguiente línea donde compartiremos la carpeta /home con permisos de lectura y escritura a los equipos dentro de la subred 192.168.1.0/24 (hay que ajustar esto a tu dirección de red) - cada segmento se separa con tabuladores:

/home    192.168.1.0/24(rw,sync,no_subtree_check)

3. Por último exportaremos la configuración NFS para que sea efectiva:

exportfs -ra

Pulse aquí para ver como montar la carpeta /home persistentemente en los clientes.

Configuración en el cliente

1. Primero editaremos el siguiente archivo para poder definir un nombre de dominio como por ejemplo iso.asir:

nano /etc/defaultdomain

Además le asignaremos a este cliente su dominio DNS correspondiente:

hostnamectl set-hostname <Hostname_Cliente.dominio.tld>

2. A continuación editaremos la configuración de NIS:

nano /etc/default/nis

Dentro de este archivo definiremos la función del cliente con las siguientes líneas:

NISSERVER=false
NISCLIENT=true

3. Ahora estableceremos un registro DNS interno por cada cliente y servidor del dominio con el siguiente archivo:

nano /etc/hosts

Definiremos con nuevas líneas lo siguiente, con una separación entre elementos con tabuladores:

<IP_Servidor>    <Hostname_Servidor.dominio.tld>    <Hostname_Servidor>
<IP_Cliente>     <Hostname_Cliente.dominio.tld>     <Hostname_Cliente>
...

4. Ahora tenemos que definir qué servidor NIS utilizará el cliente:

nano /etc/yp.conf

En la última línea hay que añadir lo siguiente:

ypserver <Hostname_Servidor.dominio.tld>

5. Ahora reiniciaremos todos los servicios y si hay fallos habrá que resolverlos:

systemctl restart rpcbind ypbind nscd

6. Una vez que el paso anterior funcione sin errores habrá que habilitar los servicios para que estén iniciados aunque se reinicie el equipo:

systemctl enable rpcbind ypbind nscd

7. Es importante configurar que el cliente sea capaz de adoptar esos usuarios y grupos desde el servicio nis con el fichero:

nano /etc/nsswitch.conf

Dentro de este fichero cambiaremos esto:

passwd:         files systemd
group:          files systemd
shadow:         files systemd
gshadow:        files systemd

A esto, añadiendo al final de cada línea la palabra nis:

passwd:         files systemd nis
group:          files systemd nis
shadow:         files systemd nis
gshadow:        files systemd nis

8. Si todo funciona deberíamos ser capaces de ver los usuarios y grupos creados en el servidor con el siguiente comando:

ypcat passwd

Como prueba podemos ejecutar lo siguiente definiendo un nombre de un usario que se haya creado en el servidor y no en el cliente:

id <Usuario_del_servidor>

Montar la carpeta /home con NFS

A continuación están los pasos guiados para montar la carpeta /home del servidor en cada cliente NIS:

1. Primero instalaremos el servicio de NFS:

apt install -y nfs-common

2. Añadiremos un montaje persistente y automático en /etc/fstab:

nano /etc/fstab

Dentro de este fichero añadiremos la siguiente línea para poder montar la carpeta del servidor en el sistema local:

<IP_servidor>:/home    /home    nfs    defaults    0    0

3. Por último, reiniciaremos el sistema para aplicar los cambios:

reboot -h now

Como resultado podremos iniciar sesión en los usuarios del sistema del servidor y ver sus carpetas con los archivos compartidos. Este sistema es útil para ver los archivos del servidor tanto en la máquina local del servidor y máquinas clientes donde se inicie sesión. 

login <Usuario_del_servidor>
pwd

Problema - NFS sobreescribe el contenido de /home

Hay que tener en cuenta que al montar una carpeta con NFS en el cliente en una ruta que ya existe y tiene contenido dentro (/home), va a provocar que el contenido local se machaque y solo se enseñe el contenido de la carpeta remota que se ha montado.

Soluciones:

  • Crear una carpeta dedicada a los usuarios de NIS como /home-nis y dejar /home para usuarios locales.
    • Ventajas: Es automático del lado del cliente ya que solo se monta una carpeta.
    • Desventajas: En cada usuario se debe cambiar la ruta de su carpeta personal.

  • Crear un script que se ejecutará cuando se vaya a montar un nuevo usuario en los clientes - explicación.
    • Ventajas: Interactivo y fácil de usar. En el servidor solo hay que compartir la carpeta /home.
    • Desventajas: Debe ejecutarse el script por cada nuevo usuario en los equipos clientes (o automatizarlo por SSH).

Solución - Script /etc/fstab

El siguiente script funciona de la siguiente manera:

  1. Crea un bucle.
  2. Al principio del bucle pide que le pases un nombre de usuario de NIS.
  3. Crea la carpeta personal si no existe.
  4. Automáticamente crea una nueva línea en /etc/fstab para montar la carpeta individual de ese usuario en el sistema.
  5. Monta las carpetas y verifica si hay errores.
  6. Repite el bucle - se puede cortar con CTRL + C.

Pasos para crear el script:

1. Primero crearemos el archivo:

nano script.sh

2. Ahora añadiremos el siguiente contenido:

#!/usr/bin/bash
while true; do
	read -p "Usuario: " usuario
	mkdir -p /home/$usuario
	echo "192.168.1.2:/home/$usuario	/home/$usuario	nfs	defaults	0	0" >> /etc/fstab
	mount -a
done

3. Le daremos al archivo permisos de ejecución:

chmod +x script.sh

4. Ejecutaremos el script:

./script.sh

Comando "getent passwd"

El siguiente comando permite ver todos los usuarios locales y remotos (NIS) gracias al archivo /etc/nsswitch.conf. Es un alternativa mejor a leer manualmente el archivo /etc/passwd y al comando ypcat passwd:

getent passwd
