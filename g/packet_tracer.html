Básico

Sintaxis de Comandos

prompt>comando argumentos

prompt>: Indica el hostname del equipo que estamos configurando, aparte de si tenemos permisos: #, o no: >.

comando: Comando a ejecutar.

argumentos: Antes de especificarlos una palabra clave o argumentos, se añade un espacio después del comando.

Reglas de Nomenclatura (Hostnames)

Reglas: Comenzar con una letra. No contener espacios. Finalizar con letra o dígito. Utilizar únicamente letras, dígitos y guiones. Tener menos de 64 caracteres de longitud. Ejemplo válido: 

Sw-Floor-1

Para poder cambiar el hostname de un equipo Cisco se usa el siguiente comando:

# hostname <Nombre>

Reglas de Contraseñas

Reglas: Tener más de 8 caracteres de longitud. Combinaciones de letras, números y símbolos. Evitar el uso de la misma en varios dispositivos. No usar palabras comunes.

Generador de contraseñas: letpass.com

Gestión de la configuración

Configuración Inicial y Ayuda

Si escribimos un comando no válido intentará resolver ese texto como DNS. Este proceso lleva tiempo y se puede saltar mediante las teclas:

CTRL + BLOQ MAYUS + 6

Habilitar permisos de EXEC privilegiado en el equipo. El prompt se modificará de > a #:

> enable

Muestra todos los comandos disponibles según donde te encuentres:

# ?

Se puede poner un comando seguido de ? para enseñar sus palabras claves y atributos disponibles. Además puedes buscar una palabra completa poniendo la(s) letra(s) por la(s) que empieza seguido de ?.

Para configurar la fecha y hora del equipo:

# clock set hora:min:seg día mes año

Mes: Debe ser el nombre con letras y en inglés.

Luego podemos ver que hora tiene configurado el equipo con:

# show clock

Gestión de Archivos de Configuración

Para ver las configuraciones del dispositivo debemos estar en la raíz del dispositivo y en modo EXEC privilegiado.


El siguiente comando nos permite ver configuraciones (interfaces, login y contraseñas) que están guardadas temporalmente y se eliminan al reiniciar el equipo:

# show running-config

Para ver la configuración que persiste en NVRAM incluso al reiniciar el equipo:

# show startup-config

Para guardar la memoria temporal en NVRAM se utiliza wr o:

# copy running-config startup-config

Para eliminar la memoria temporal se utiliza:

# reload

Por último, para eliminar la memoria NVRAM se utiliza:

# erase startup-config

Sistema Operativo

Sistema Operativo IOS

Para cambiar de la variable de entorno BOOT la ruta al archivo de imagen se utiliza el siguiente comando:

(config)# boot system flash:/<SistemaArchivos>/<NombreArchivoIOS>.bin

El cargador de arranque proporciona acceso al switch si no se puede usar el sistema operativo debido a la falta de archivos de sistema o al daño de estos. Permite acceso a los archivos almacenados en la memoria flash. Dentro de este aparecerá el prompt switch:.

Eliminar toda la configuración del sistema

Eliminaremos la configuración de arranque y el fichero de la configuración de VLANs:

# erase startup-conf
# delete vlan.dat
# write erase

Cuando pregunte si estamos seguros debemos responder con y seguido de ENTER.

Herramientas de Conectividad

Herramientas de Conectividad (Ping/SSH/Telnet)

Enviar un paquete ICMP a un host de destino para verificar si responde:

ping <IpDestino>

Conectarnos mediante Secure Shell a un host remoto:

ssh -l <Usuario> <IpDestino>

Conectarnos mediante Telnet a un host remoto:

telnet (-l <Usuario>) <IpDestino>

El parámetro -l en el Command prompt de los PCs no funciona, solo hay que definir la IP.

Nota - TELNET y SSH:

Si en el router o switch que queremos conectarnos no tiene en las líneas vty 0 15 un password y login, el destino rechazará la conexión automáticamente.

⚠️ Nota - Problema de seguridad en TELNET:

El tráfico de Telnet se envía en texto plano, incluyendo usuario y contraseña, lo que lo hace vulnerable a ataques man-in-the-middle. Demostración en Packet Tracer:

1. Entra en el Command Prompt de un PC e inicia una conexión:

C:\> telnet (-l <Usuario>) <IpDestino>

2. Antes de ingresar la contraseña, cambia Packet Tracer de Realtime a Simulation.

3. Se puede apreciar que por cada letra que tenga la contraseña se crea un paquete de parte del cliente. Abre a uno desde la sección Event List.

4. En la nueva ventana pasa al modo Outbound PDU Details y baja hasta el apartado TELNET. Se puede ver que en cada paquete se enseña la letra enviada:

TELNET DATA:C

Filtrar la salida de un comando

Sintaxis para filtrar:

<Comando> | <Filtro> <Palabra(s) a filtrar>

FILTROS:

Para filtrar una sección entera mediante un nombre como vty:

show running-config | section line vty

Esto devolvería toda la sección vty incluyendo su configuración como:

Para filtrar solo las líneas que contengan una palabra como up:

show ip interface brief | include up

Esto devolvería toda las líneas que contengan la palabra up:

Comandos de Verificación

Comandos de Verificación > Importante

El comando show solo funciona si se ejecuta fuera de (config)# como el siguiente ejemplo:

# show ip interface brief

Si se desea ver el comando dentro de la configuración hay que añadir la palabra do al principio del comando como el siguiente ejemplo:

(config)# do show ip interface brief

Tenga en cuenta que al usar do no se podrá usar el autocompletado de opciones del comando.

Comandos de Verificación > Interfaces

Para ver todas las configuraciones de las interfaces del dispositivo en IPv4:

show ip interface brief
show ipv6 interface brief

Enseña de cada interfaz el nombre, IP, Si está reconocida por el sistema (OK?), estado y protocolo. Ejemplo:

Router# show ip interface brief
Interface              IP-Address      OK? Method Status                Protocol
FastEthernet0/0        192.168.1.1     YES manual up                    up
FastEthernet0/1        unassigned      YES unset  administratively down down
Vlan1                  unassigned      YES unset  administratively down down

Para ver todas las estadísticas y estado de todas las interfaces del equipo:

show interfaces

Este comando es útil para detectar problemas comunes de medios. Una de las partes más importantes de esta salida es la visualización de la línea y el estado del protocolo de enlace de datos.

También se puede ver las estadísticas y estado de una interfaz en específico:

show interfaces <Interfaz>

Esto puede darnos información sobre errores de tramas gigantes, colisiones, runts, desbordamiento, etc.

Los siguientes comandos muestran información de IPv4 o IPv6 de todas las interfaces:

show ip interface brief
show ipv6 interface brief

Los siguientes comandos muestran información de IPv4 o IPv6 de una interfaz en específico:

show ip interface <Interfaz>
show ipv6 interface <Interfaz>

En ipv6 se muestra todas las direcciones IPv6 que pertenecen a la interfaz (Joined group address(es)).

Se puede ver los comandos aplicados a una interfaz específica que se están aplicando actualmente:

show running-config interface <Interfaz>

Para ver la tabla ARP IPv4:

show ip arp

Para ver la tabla de vecinos IPv6:

show ipv6 neighbors

Para ver la tabla MAC se puede usar las 2 siguientes formas:

show mac address-table
show mac-address-table

Comandos de Verificación > Tabla de Rutas

Para ver la tabla de enrutamiento en IPv4:

show ip route

Para ver la tabla de enrutamiento en IPv6:

show ipv6 route

Configuración básica

Navegación y Configuración de Interfaces

Podemos entrar en modo de configuración global del dispositivo de 2 formas:

# configure terminal
# config t

Podemos configurar un puerto de switch o una interfaz de red de router (solo si entramos en config anteriormente):

(config)# interface <Interfaz><Número>

Interfaz y Número: Podemos especificar un nombre completo como FastEthernet0/1 o su abreviatura como fa0/3. 0/3 Significa que de la interfaz 0, se elige el puerto 3.

Podemos asignar una descripción a cada interfaz:

(config-if)# description <Descripción>

Para configurar parámetros de consola co, SSH y Telnet vty o acceso auxiliar:

(config)# line <co/vty> <Número/Rango>

Número: El número empieza desde 0 e identifica el puerto a configurar. Rango: Si se quiere configurar varias líneas a la vez, como las 16 líneas de vty se usa un rango como: 0 15, con un espacio entre ambos números.

Habilitar una interfaz después de realizar un cambio:

(config-if)# no shutdown

O deshabilitarla:

(config-if)# shutdown

Podemos salir de cualquier modo de configuración al anterior. Ej. Si nos encontramos en (config-if) y enviamos este comando pasamos a (config):

(config-if)# exit
(config)# exit
# exit
> <Comandos>

O podemos salir de cualquier modo de configuración al modo inicial:

(config-if)# end
# <Comandos>

Direccionamiento IPv4 e IPv6

Si es un switch hay que configurar IPs mediante VLANs se usa (la VLAN por defecto es 1):

(config)# interface vlan <VLAN_ID>

Podemos asignar a una interfaz una IPv4 mediante:

(config)# interface <Interfaz><Número>
(config-if)# ip address <IPv4> <Máscara>
(config-if)# no shutdown

Nota: Hay que recordar poner la configuración del default-gateway y name-server.

O además una IPv6:

(config)# interface <Interfaz><Número>
(config-if)# ipv6 address <IPv6/Máscara>
(config-if)# no shutdown

Si se trata de una dirección IPv6 LLA fe80::/10 se modifica el comando a:

(config-if)# ipv6 address <LLA_IPv6> link-local

Default Gateway (Switch)

Para poner la IP del gateway por defecto hay que ponerlo en la configuración global:

(config)# ip default-gateway <IpGateway>

Enrutamiento Estático

Para añadir una ruta a la que un router pueda acceder de manera manual:

(config)# ip route <IP_RedRemota> <Máscara_RedRemota> <IP_PróximoRouter>

Nota: Si la dirección de red o una IP cambia, debe modificarse la ruta manualmente.

Servicios DNS

Asignar un nombre de dominio (ej. pargapondal.inf):

(config)# ip domain-name <Dominio>

Desactivar la búsqueda de DNS:

(config)# no ip domain-lookup

Para asignar un servidor DNS a un switch o router hay que ponerlo en la configuración global:

(config)# ip name-server <IP_Servidor_DNS>

Seguridad

Filtro de Protocolos (Transport Input)

Podemos permitir ciertos protocolos de acceso remoto con:

(config)# line vty 0 15
(config-line)# transport input <ssh telnet>

Podemos permitir tanto ssh y telnet o solo uno de los dos. Se separan mediante espacios.

Seguridad Básica y Banners

Poner una contraseña en líneas de acceso:

(config)# line <console/vty> <NumPuerto/Rango>
(config-line)# password <Contraseña>

Para poner una contraseña al comando enable (texto plano):

(config)# enable password <Contraseña>

Para ocultar la contraseña:

(config)# service password-encryption

Aunque oculta el contenido, se puede descifrar fácilmente debido a su débil cifrado con herramientas como firewall.cx.

Para poner una contraseña al comando enable (encriptando con MD5):

(config)# enable secret <Contraseña>

Si se tenía enable password anteriormente se ignorará y solo se usará enable secret.

Aplicar una política de usuario y contraseña: authentication, o solo contraseña: local:

(config-line)# login <authentication/local>

Poner un banner que se enseñará en la página de login:

(config)# banner motd <Carácter>

Carácter: Indicamos un carácter que no se vaya a repetir en el texto. Después de ejecutarlo pondrá un diálogo para poner un texto personalizado que verán al iniciar sesión. Acabaremos el comando con el carácter especificado y luego ENTER.

Bloquear un rango de puertos no usados:

(config)# interface range <Interfaz/Rango>, <Interfaz/Rango>, ...

Interfaz/Rango: Podemos seleccionar varios rangos de interfaces y aplicar los mismos cambios a todos. Ejemplo: F0/2-24, G0/2 (F0/2 a F0/24 y G0/2)

(config-if-range)# shutdown

Gestión de Usuarios y Privilegios

Para crear un usuario sin privilegios:

(config)# username <Nombre_Usuario> secret <Contraseña>

Para crear un usuario con privilegios:

(config)# username <Nombre_Usuario> privilege <TipoPrivilegio> secret <Contraseña>

TipoPrivilegio: Número del 0 al 15 que indica el nivel de privilegios que tendrá el usuario:

- Del 2 al 14 son usuarios con permisos parciales.
- El 15 equivale al comando enable.
- El 1 equivale a un usuario normal.
- El 0 equivale a un acceso muy restringido.

Para aplicar una directiva de autentificación por usuario y contraseña:

(config)# line vty 0 15
(config-if)# login local

Políticas de Seguridad (Bloqueos)

Para aplicar una longitud mínima a cada contraseña que se cree desde este momento (no se aplica a contraseñas ya creadas):

(config)# security passwords min-length <NúmeroMínimo>

Para cerrar la sesión de un usuario después de x minutos:

(config)# line vty 0 15
(config-if)# exec-timeout <Minutos>

Bloquear a cualquier persona durante x minutos que no pueda iniciar sesión después de x intentos dentro de un período de x minutos:

(config)# login block-for <SegundosBloqueo> attempts <Intentos> within <PeriodoMinutos>

Configuración Servidor SSH

Pasos:

1. Antes de nada hay que verificar si el switch soporta realizar funciones criptográficas. Para ello debe de ejecutar este comando sin errores: 

# show ip ssh

2. Poner un hostname único y diferente al de por defecto.

3. Asignar un nombre de dominio (ej. pargapondal.inf):

(config)# ip domain-name <Dominio>

4. Generar una clave de 4096 Bytes u otro tamaño con el algoritmo RSA:

(config)# crypto key generate rsa general-keys modulus 4096

Permitir solo el tráfico SSH en las líneas de acceso remoto:

(config)# line vty 0 15
(config-if)# transport input ssh

Desde el momento de poner el comando se denegará conexiones telnet, solo admitirá SSH. Si se tenía alguna conexión telnet existente no la cierra.

Hay que recordar de poner login local si se va a permitir el acceso con nombre de usuario y contraseña.

🛡️ Para reforzar la seguridad, se puede forzar al switch a realizar únicamente conexiones por SSH v2.0:

(config)# ip ssh version 2

Configuración específica

Configuración de full-duplex en una interfaz

Primero entraremos a la interfaz deseada y asignaremos el modo full-duplex:

(config)# interface <Interfaz>
(config-vlan)# duplex full

Configuración de velocidad en una interfaz

Primero entraremos a la interfaz deseada y asignaremos la velocidad en Mbps:

(config)# interface <Interfaz>
(config-vlan)# speed <VelocidadEnMbps>

Configuración de Auto-MDIX en una interfaz

Primero entraremos a la interfaz deseada y asignaremos que funcione como Auto-MDIX:

(config)# interface <Interfaz>
(config-vlan)# mdix auto

Podemos comprobar que se haya aplicado Auto-MDIX en la interfaz especificada con el comando:

# show controllers ethernet-controller <Interfaz> phy | include MDIX

Si es correcto debería de devolver la siguiente línea:

Auto-MDIX    :  On    [AdminState=1    Flags=0x00052248]

Creación de una interfaz loopback

Una interfaz de loopback en un router no permite conectar otros dispositivos y sirve para que una interfaz siempre esté encendida (siempre que el router esté en funcionamiento). Por ejemplo, se puede usar con fines de prueba, como la prueba de procesos de routing interno, mediante la emulación de redes detrás del router.

Para crear una, simplemente se ejecuta el siguiente comando seguido de la asignación de una IP:

(config)# interface loopback <Número>
(config-if)# ip address <IP> <Máscara>

VLANs

VLAN > Creación

Primero crearemos la VLAN con el siguiente comando, indicando su número de VLAN:

(config)# vlan <VLAN_ID>

Luego podremos asignarle un nombre personalizado con:

(config-vlan)# name <NombreVLAN>

VLAN > Eliminar la configuración de todas las VLANs

Con el comando delete se eliminará el fichero de configuración de las VLANs:

(config)# delete flash:vlan.dat

Otra forma abreviada es quitando el dispositivo flash de la ruta:

(config)# delete vlan.dat

VLAN > Visualización de puertos asignados

Con el siguiente comando se puede ver qué VLANs se asignaron a ciertos puertos:

# show vlan brief

Por defecto se puede ver que todas las interfaces del switch se asignaron a la VLAN 1 (predeterminada).

Con el siguiente comando se muestra la información de una VLAN específica:

# show vlan id <VLAN_ID>

VLAN > Visualización de la configuración

Hay un comando muy útil que indica de una interfaz su modo de funcionamiento, VLANs asignadas, permitidas, etc.:

# show interfaces <Interfaz> switchport

Esto devolverá algo parecido a:

Switchport: Enabled
Administrative Mode: trunk
Operational Mode: trunk
Administrative Trunking Encapsulation: dot1q
Operational Trunking Encapsulation: dot1q
Access Mode VLAN: 1 (default)
Trunking Native Mode VLAN: 99 (VLAN0099)
Voice VLAN: none
Trunking VLANs Enabled: 10,20,30,99

VLAN > Asignación de una IP

Simplemente entraremos a la interfaz con interface y le asignaremos la configuración necesaria:

(config)# interface vlan <VLAN_ID>
(config-if)# ip address <IP> <MáscaraPunteada>
(config-if)# no shutdown

Para verificarlo se puede ejecutar el siguiente comando:

# show ip route

Tendría que aparecer una línea como la siguiente, que indica que si se quiere llegar a la subred <Direccion_Red>/<Máscara>, el dispositivo enrutará el paquete hacia la interfaz lógica Vlan<VLAN_ID>. Luego, internamente, el sistema enviará el paquete por el puerto físico correspondiente a la VLAN: 

C    <Direccion_Red>/<Máscara> is directly connected, Vlan<VLAN_ID>

VLAN > Modo Acceso > Configuración de una interfaz

Entraremos a la interfaz, lo configuraremos el modo acceso y le asignaremos una VLAN:

(config)# interface <Interfaz>
(config-if)# switchport mode access
(config-if)# switchport access vlan <VLAN_ID>

Si se quiere eliminar el modo acceso directamente:

(config-if)# no switchport mode access

VLAN > Modo Acceso > Restablecer

Entraremos a la interfaz, y le quitaremos de switchport la vlan:

(config)# interface <Interfaz>
(config-if)# no switchport access vlan

Switchport

Modos de interfaz negociados (Switchport)

El comando switchport mode tiene opciones adicionales para negociar el modo de interfaz. Sintaxis del comando:

Switch(config)# switchport mode {  access | dynamic {  auto | desirable } |  trunk }
Modos de switchport 
access
 Pone la interfaz (puerto de acceso) en modo permanente de no trunking y negocia para convertir el enlace en un enlace no troncal. La interfaz se convierte en una interfaz no troncal, independientemente de si la interfaz vecina es una interfaz troncal. 
dynamic auto
 La interfaz se convierte en una interfaz de enlace troncal, si la interfaz vecina está configurado en modo troncal o deseable. 
dynamic desirable
 La interfaz se convierte en una interfaz de enlace troncal, si la interfaz vecina está configurado en modo automático troncal, deseable o dinámico. 
trunk
 El puerto queda configurado en modo troncal de forma permanente y negocia para que el enlace se convierta en una conexión troncal. La interfaz se convierte en una interfaz de enlace troncal, incluso si la interfaz vecina no es una interfaz troncal.

Verificación del modo de DTP

# show dtp interface <Interfaz>

Spanning-tree

Spanning-tree > Asignar un switch como raíz

Dentro del switch que queramos definir como raíz ejecutaremos lo siguiente:

(config)# spanning-tree vlan <VLAN_ID> root primary

VLAN_ID: La VLAN en la que este switch será raíz.

Para verificarlo ejecutaremos el siguiente comando:

(config)# do show spanning-tree vlan <VLAN_ID>

En el apartado Root ID > Priority se puede observar que el valor es 4096 menor que la prioridad del switch que anteriormente actuaba como raíz, lo que confirma que el nuevo switch ha sido elegido como raíz.

Nota Teórica: Este comando no fija una prioridad estática simple. El switch verifica la prioridad del raíz actual y configura su propia prioridad para ser 4096 puntos menor (si es posible) para ganar la elección. Si el raíz actual tiene 24576, este switch bajará a 20480.

Spanning-tree > Asignar un switch como secundario

Dentro del switch que queramos definir como secundario ejecutaremos lo siguiente:

(config)# spanning-tree vlan <VLAN_ID> root secondary

VLAN_ID: La VLAN en la que este switch será secundario.

Para verificarlo ejecutaremos el siguiente comando:

(config)# do show spanning-tree vlan <VLAN_ID>

En el apartado Bridge ID > Priority se puede observar que el valor no es menor que el de Root ID > Priority pero si es menor que el de otros switches en esa VLAN.

Nota Teórica: Este comando fija la prioridad exactamente en 28672. Si todos los demás switches tienen la prioridad por defecto (32768), este switch "secundario" se convertirá en el Raíz (porque 28672 < 32768). Solo actuará como secundario si existe otro switch con prioridad "Primary" (ej. 24576).

Spanning-tree > Conceptos de Prioridad

La prioridad del bridge se compone de:
Prioridad Total = Prioridad Base + ID de VLAN

Los valores cambian en bloques de 4096. Los valores típicos son:

Por defecto 32768 + VLAN
Root Secondary 28672 + VLAN
Root Primary 24576 + VLAN (o 4096 menos que el raíz actual)

Nota de Desempate: Si dos switches tienen la misma prioridad (ej. ambos por defecto), gana el que tenga la dirección MAC más baja.

EtherChannel - LACP

EtherChannel > LACP > Configuración

Antes de realizar los siguientes pasos, asegúrese de que las VLANs ya estén creadas y que las demás interfaces necesarias estén correctamente configuradas en ambos switches.

Paso 1. Configurar el EtherChannel:

(config)# interface Po<Número>
(config-if)# switchport mode trunk
(config-if)# ...

En esta interfaz lógica se debe aplicar exactamente la misma configuración que se usaría en las interfaces físicas que formarán el EtherChannel.

El número es un identificador que representa al EtherChannel y debe coincidir con el que se usará en los pasos posteriores.

Paso 2. Aplicar la misma configuración del paso anterior en las interfaces físicas que se agruparán en el EtherChannel:

(config)# interface range <Interfaz1,Interfaz2,...>
(config-if-range)# switchport mode trunk
(config-if-range)# ...

Todas las interfaces deben tener configuraciones idénticas para que el EtherChannel funcione correctamente.

Paso 3. Apagar las interfaces:

(config)# interface range <Interfaz1,Interfaz2,...>
(config-if-range)# shutdown

Este paso evita inconsistencias o errores durante la creación del EtherChannel.

Paso 4. Agregar las interfaces al EtherChannel:

(config)# interface range <Interfaz1,Interfaz2,...>
(config-if-range)# channel-group <Número> mode active

El número debe coincidir con el identificador del EtherChannel configurado en el paso 1.

El modo active indica que se utilizará LACP para la negociación del enlace.

Importante: Antes de continuar, asegúrese de haber ejecutado correctamente todos los pasos anteriores en ambos switches.

Paso 5. Encender las interfaces:

(config)# interface range <Interfaz1,Interfaz2,...>
(config-if-range)# no shutdown

Una vez habilitadas, las interfaces deberían formar el EtherChannel correctamente.

Nota: Una vez creado el EtherChannel, cualquier cambio que se vaya a hacer a las interfaces se debe hacer solo al grupo Po<Número> mediante el comando:

(config)# interface Po<Número>
(config-if)# ...

EtherChannel > LACP > Verificación

Para verificar si el EtherChannel se creó correctamente hay que ejecutar el siguiente comando en ambos switches:

# show etherchannel summary

Importante: En el apartado "Ports" debe aparecer una P como en el siguiente ejemplo:

# show etherchannel summary
    Number of channel-groups in use: 1
    Number of aggregators:           1

    Group  Port-channel  Protocol    Ports
    ------+-------------+-----------+------------------
    1      Po1(SU)           LACP   Gig0/1(P) Gig0/2(P)

Otra manera de verificarlo es mirar si la interfaz port-channel está encendida:

# show interfaces port-channel <Número>

El número es un identificador que representa al EtherChannel.

EtherChannel > LACP > Eliminación de un EtherChannel existente

Paso 1. Entrar a las interfaces y eliminar el channel-group:

(config)# interface range <Interfaz1,Interfaz2,...>
(config-if-range)# no channel-group <Número>

Paso 2. Eliminar el port-channel:

(config)# no interface port-channel <Número>

El número del port-channel es el mismo que se le asignó a la interfaz EtherChannel Po<Número>.

VLAN Trunking Protocol

VTP > Configuración en el switch servidor

1. Meteremos al switch en un dominio VTP:

(config)# vtp domain <Nombre_Dominio>

2. Asignaremos una contraseña necesaria para que otros switches tengan que introducirla para conectarse al dominio:

(config)# vtp password <Contraseña>

3. Asignaremos al switch que actúe en modo servidor:

(config)# vtp mode server

4. Por seguridad, asignaremos que el switch use solo VTP versión 2:

(config)# vtp version 2

⚠️ Advertencia de Seguridad (Dominio NULL):

Por defecto, un switch no tiene nombre de dominio (es NULL). Si conectas un switch nuevo a la red sin configurar el dominio primero, este adoptará automáticamente el nombre del dominio del primer paquete VTP que reciba. Esto puede causar que el switch se una a la red equivocada y altere las VLANs.

VTP > Meter a un cliente en un dominio

1. Meteremos al switch en el mismo dominio VTP que el servidor:

(config)# vtp domain <Nombre_Dominio>

2. Asignaremos la contraseña que se definió en el servidor:

(config)# vtp password <Contraseña>

3. Por seguridad, asignaremos que todo el switch use solo VTP versión 2:

(config)# vtp version 2

4. Definir en qué modo va a trabajar el switch:

(config)# vtp mode <client/transparent>

client: En este modo solo se podrá cambiar la lista de VLANs cuando se reciba comunicaciones de otros equipos.
transparent: En este modo el dispositivo actuará como un dispositivo aislado, sin compartir sus VLANs con el resto de dispositivos.

VTP > Ver la configuración

Para verificar el estado de VTP y la configuración usaremos el siguiente comando:

# show vtp status

Esto mostrará todas las configuraciones anteriores menos la contraseña, además de un Configuration Revision que es muy importante al indicar la cantidad de cambios que se realizaron en este switch (por defecto es 0).

En cualquier otro switch que tenga un Configuration Revision menor al switch servidor, tendrá que actualizar su lista de VLANs.

VTP > Ver la contraseña del dominio

Cualquier switch de un mismo dominio debe tener la misma contraseña VTP que indique el siguiente comando:

# show vtp password

VTP > Resetear la Revisión (Seguridad)

Importante: Antes de conectar un switch a la red, debes asegurar que su "Configuration Revision" sea 0.

Si conectas un switch con una revisión mayor a la del servidor, este sobrescribirá y borrará las VLANs de toda la red.

Método 1 - Cambiar el modo a Transparente y volver:

Al pasar a modo transparente, el contador de revisión se reinicia a 0.

(config)# vtp mode transparent
(config)# vtp mode client

Método 2 - Borrar la base de datos de VLANs:

# delete vlan.dat
# reload

Hot Standby Router Protocol

HSRP > Configuración de una IP

En cada switch de capa 3 o router en el que se vaya a compartir una IP en caso de que un dispositivo caiga, realizaremos los siguientes pasos:

1. Entraremos la interfaz física o lógica (vlan) a la que configuraremos la IP en HSRP:

(config)# interface <Interfaz>

2. Le asignaremos una IP diferente a la que se va a configurar con HSRP:

(config-if)# ip address <IP_Secundaria>

3. Asignaremos que se use solo la versión 2 de HSRP:

(config-if)# standby version 2

4. Ahora meteremos a la interfaz en un grupo y le asignaremos que IP compartirá con otro(s) dispositivo(s):

(config-if)# standby <Grupo> ip <IP_HSRP>

Grupo: Número que vincula lógicamente a varios equipos para que funcionen como un único equipo virtual.
IP_HSRP: La dirección IPv4 que compartirá con otros dispositivos en el mismo grupo.

5. Por último encenderemos la interfaz si no se ha hecho anteriormente:

(config-if)# no shutdown

HSRP > Configuración de la prioridad

1. Entraremos la interfaz física o lógica (vlan) a la que configuraremos la prioridad:

(config)# interface <Interfaz>

2. Le asignaremos una prioridad numérica con el siguiente comando:

(config)# standby <Grupo> priority <Prioridad>

Grupo: Número que vincula lógicamente a varios equipos para que funcionen como un único equipo virtual.
Prioridad: Número de 0 a 255 que indicará qué prioridad tenga el dispositivo. Cuanto mayor sea, mayor preferencia tendrá para ser el dispositivo activo.

Nota: No hace falta definir que tiene prioridad 100 ya que es la prioridad por defecto que tienen todos los switches.

HSRP > Configuración de preempt

Con preempt podemos forzar el cambio de cesión de IP de un dispositivo a otro si hay un cambio de prioridad.

1. Entraremos la interfaz física o lógica (vlan) a la que configuraremos el modo:

(config)# interface <Interfaz>

2. Le asignaremos que funcione en modo preempt:

(config)# standby <Grupo> preempt

Grupo: Número que vincula lógicamente a varios equipos para que funcionen como un único equipo virtual.

Nota: No hace falta poner preempt en el switch que no será el activo.

En el comando show standby brief debería aparecer la interfaz que el modo preempt está activo (P).

HSRP > Verificación de la configuración

Para verificar el estado de HSRP y la configuración de cada interfaz con IP compartida usaremos el siguiente comando:

# show standby

Este comando muestra detalles que el comando con brief omite, como la MAC virtual activa, los temporizadores por defecto (Hello 3s, Hold 10s) y el contador de cambios de estado del router.

También se puede ver en formato de resumen:

# show standby brief

Seguridad - Spanning-tree

Spanning-tree > Activar Port-Fast

Primero entraremos en la interfaz deseada y activaremos mediante spanning-tree el modo portfast:

(config)# interface <Interfaz>
(config-if)# spanning-tree portfast

Esto evitará que la interfaz espere alrededor de 50 segundos para empezar a aceptar tramas.

Importante: Debe ponerse solo en puertos que estén en modo acceso.

Además podemos habilitar portfast por defecto en todos los puertos del switch que están en modo acceso:

(config)# spanning-tree portfast default

Spanning-tree > Activar BPDU-Guard

Primero entraremos en la interfaz deseada y activaremos mediante spanning-tree el modo bpduguard:

(config)# interface <Interfaz>
(config-if)# spanning-tree bpduguard enable

Esto pondrá la interfaz en estado err-disable si recibe una BPDU, para evitar que se conecte un switch y provoque bucles en la red.

Importante: Debe ponerse solo en puertos que estén en modo acceso y en portfast ya que en los puertos en modo troncal suelen circular mensajes STP, ya que en los puertos en modo troncal suelen circular BPDUs, lo que provocaría que la interfaz entrase en err-disable.

Además podemos habilitar BPDU-Guard por defecto en todos los puertos del switch que están en modo acceso:

(config)# spanning-tree portfast bpduguard default

Siempre hay que activar portfast si se usa BPDU-Guard.

Spanning-tree > Verificación

Con el siguiente comando podremos ver toda la configuración relacionada con STP:

# show running-config | begin span

También podemos ver un resumen de la configuración de STP mirando si portfast o BPDU-Guar está habilitado:

# show spanning-tree summary

Seguridad - Problemas de reconocimiento

(Des)habilitar el protocolo CDP

CDP es un protocolo habilitado por defecto en todos los switches de Cisco. Puede detectar automáticamente otros dispositivos con CDP habilitado y ayudar a configurar automáticamente la conexión.

Problema de seguridad: La información de CDP se envía por los puertos con CDP habilitado en transmisiones periódicas sin encriptar, incluyendo datos de configuración del switch. Esto es un problema ya que un atacante puede recibir estos datos y detectar vulnerabilidades según la versión del switch que envió el mensaje.

Como solución podemos deshabilitar globalmente el protocolo CDP:

(config)# no cdp run

Para volver a habilitar el protocolo CDP globalmente podemos ejecutar:

(config)# cdp run

Otra opción sería deshabilitarlo en una interfaz específica:

(config)# interface <Interfaz>
(config-if)# no cdp enable

Para volver a habilitar el protocolo CDP en esa interfaz podemos ejecutar:

(config-if)# cdp enable

(Des)habilitar el protocolo LLDP

LLDP también es vulnerable a los ataques de reconocimiento como el protocolo CDP.

Como solución podemos deshabilitar globalmente el protocolo LLDP:

(config)# no lldp run

Para volver a habilitar el protocolo LLDP globalmente podemos ejecutar:

(config)# lldp run

Otra opción sería deshabilitarlo en una interfaz específica:

(config)# interface <Interfaz>
(config-if)# no lldp transmit
(config-if)# no lldp receive

Para volver a habilitar el protocolo LLDP en esa interfaz podemos ejecutar:

(config-if)# lldp transmit
(config-if)# lldp receive

Seguridad - Puertos

Deshabilitar puertos no usados

Una buena práctica es apagar todos los puertos que no se utilicen y así evitar que, si alguien intenta conectar un dispositivo no autorizado, el switch no envíe ningún tipo de tráfico a ese puerto en específico: 

(config)# interface range <Rango_de_Interfaces>
(config-if)# shutdown

En caso de que queramos conectar un dispositivo podemos encender el puerto de nuevo:

(config-if)# no shutdown

Asignar puertos no usados a la VLAN BlackHole

Es buena práctica crear una vlan llamada BlackHole y meter dentro todos los puertos que no se vayan a usar:

(config)# vlan <Número_VLAN_Sin_Usar>
(config-vlan)# name BlackHole
(config-vlan)# exit
(config)# interface range <Interfaces_Sin_Usar>
(config-if)# switchport mode Access
(config-if)# switchport access vlan <Número_VLAN_BlackHole>

Seguridad - Port Security

Habilitar la seguridad de puertos

La seguridad de puertos va dirigida solo a puertos en modo acceso, y se puede activar de la siguiente forma:

(config)# interface <Interfaz>
(config-if)# switchport mode access
(config-if)# switchport port-security

Nota: Si un puerto activo está configurado con port-security y hay más de un dispositivo conectado a ese puerto, el puerto pasará al estado de error desactivado.

Seguridad de puertos > Limitar MACs

Con port-security podemos limitar la cantidad máxima de MACs que puede haber en un puerto, siendo de 1 hasta 8192:

(config)# interface <Interfaz>
(config-if)# switchport port-security maximum <Cantidad_Máxima_MACs>

Además, podemos fijar las direcciones MAC estáticas que si podrán tener acceso al puerto:

(config)# interface <Interfaz>
(config-if)# switchport port-security mac-address <Dirección_MAC>

Estas MACs que se aprendan no persisten en la configuración si se reinicia el equipo, por ello podemos definir que se guarde en NVRAM:

(config)# interface <Interfaz>
(config-if)# switchport port-security mac-address sticky

Seguridad de puertos > Caducidad de las MACs

Podemos definir que las MACs que se guardan tengan un tiempo de caducidad hasta que se eliminen de la memoria. Para las MACs dinámicas (aprendidas automáticamente) podemos configurarlo de 2 formas:

1. Caducidad absoluta - se eliminan después del tiempo de caducidad aunque estén activas:

(config)# interface <Interfaz>
(config-if)# switchport port-security aging type absolute
(config-if)# switchport port-security aging type time <Tiempo_En_Minutos>

2. Caducidad por inactividad - se eliminan solo si no transmiten tráfico durante el tiempo de caducidad:

(config)# interface <Interfaz>
(config-if)# switchport port-security aging type inactivity
(config-if)# switchport port-security aging type time <Tiempo_En_Minutos>

Para las MACs estáticas (configuradas manualmente):

(config)# interface <Interfaz>
(config-if)# switchport port-security aging static
(config-if)# switchport port-security aging type time <Tiempo_En_Minutos>

Por defecto no caducan, pero se puede habilitar el envejecimiento.

Seguridad de puertos > Modos de violación

Podemos definir que pasará si ocurre una violación de un puerto con 3 modos diferentes:

1. Que la interfaz se apague inmediatamente, generar una alerta del sistema y que se incremente el contador de violaciones:

(config)# interface <Interfaz>
(config-if)# switchport port-security violation shutdown

2. Que se descarten todos los paquetes desde y hacia esa interfaz, generar una alerta del sistema y que se incremente el contador de violaciones:

(config)# interface <Interfaz>
(config-if)# switchport port-security violation restrict

3. Que se descarten todos los paquetes desde y hacia esa interfaz sin generar ninguna alerta ni incrementar el contador de violaciones:

(config)# interface <Interfaz>
(config-if)# switchport port-security violation protect

Es el modo menos seguro de los tres.

Seguridad de puertos > Habilitar un puerto bloqueado

Si una interfaz fue bloqueada y no se configuró sticky, podemos habilitarla de nuevo:

(config)# interface <Interfaz>
(config-if)# shutdown
(config-if)# no shutdown

En caso de que sí se haya configurado sticky tendremos que quitar las direcciones MAC de la memoria:

En el siguiente comando tendremos que localizar las MACs que se guardaron en memoria (sticky):

# show run interface <Interfaz>

En el siguiente comando borraremos las líneas exactas guardadas en memoria definiendo la palabra "no" al principio:

(config-if)# no switchport port-security mac-address sticky <Dirección_MAC_A_Quitar>

Seguridad de puertos > Verificación

El siguiente comando muestra un resumen de cada interfaz con sus MACs aprendidas, el máximo y si está en modo de violación:

# show port-security

Para mostrar todas las direcciones MAC seguras que se configuran manualmente o se aprenden dinámicamente:

# show port-security address

Para verificar que una interfaz entró en modo de violación y ver su contador de violaciones podemos usar el siguiente comando:

# show port-security interface <Interfaz>

Podemos ver de una interfaz toda la configuración asignada y las MACs aprendidas en modo sticky:

# show run interface <Interfaz>

Seguridad - Ataques de VLAN

VLAN > Mitigar ataques de salto

Un atacante puede aprovechar la negociación automática de Dynamic Trunking Protocol para simular ser un switch y conseguir que el puerto negocie un enlace trunk. De esta forma podría recibir tráfico de varias VLANs.

Esto ocurre porque normalmente un equipo está conectado a un puerto en modo acceso, que solo debería tener acceso a una VLAN.

Para solucionarlo tendremos que definir los puertos del switch en modo acceso y asignarle una VLAN fija, además de deshabilitar la negociación DTP:

(config)# interface <Interfaz>
(config-if)# switchport mode access
(config-if)# switchport access vlan <Número_VLAN>
(config-if)# switchport nonegotiate

En enlaces troncales también deberíamos definir una VLAN nativa distinta a la 1, permitir solo las VLANs con tráfico y quitar la negociación de DTP:

(config)# interface <Interfaz>
(config-if)# switchport mode trunk
(config-if)# switchport trunk native vlan <Número_VLAN_Nativa>
(config-if)# switchport trunk allowed vlan <VLANs_Permitidas>
(config-if)# switchport nonegotiate

Seguridad - Ataques de DHCP

DHCP Snooping > Configuración

Para habilitar DHCP Snooping lo haremos en la configuración global:

(config)# ip dhcp snooping

Además podemos definir en qué VLANs se realizará la inspección:

(config)# ip dhcp snooping vlan <VLANs_A_Inspeccionar>

En las interfaces de confianza se debe definir explícitamente el siguiente comando:

(config)# interface <Interfaz>
(config-if)# ip dhcp snooping trust

Para limitar en una interfaz la cantidad de mensajes de descubrimiento de DHCP:

(config)# interface <Interfaz>
(config-if)# ip dhcp snooping limit rate <Número_Límite>

DHCP Snooping > Verificación

Con el siguiente comando podemos ver toda la configuración que se haya asignado, además de un esquema de todas las interfaces:

# show ip dhcp snooping

Para ver todas las concesiones DHCP (IP y MAC) en cada interfaz podemos usar el siguiente comando:

# show ip dhcp snooping binding

Seguridad - Ataques de ARP

DAI > Configuración

Para configurar DAI, se requiere que funcione la tabla de enlace de inspección DHCP. Por ello tendremos que habilitarlo:

(config)# ip dhcp snooping

A continuación habilitaremos en la VLAN correspondiente la inspección de DHCP y ARP:

(config)# ip dhcp snooping vlan <Número_VLAN>
(config)# ip arp inspection vlan <Número_VLAN>

Debemos configurar que los enlaces troncales sean puertos confiados/trust:

(config)# interface <Interfaz_Troncal>
(config-if)# ip dhcp snooping trust
(config-if)# ip arp inspection trust

Hay 3 tipos de validaciones de ARP:

1. Por la mac de origen - comprueba la dirección MAC de origen en el encabezado de Ethernet con la dirección MAC del remitente en el cuerpo ARP:

(config)# ip arp inspection validate src-mac

2. Por la mac de destino - comprueba la dirección MAC de destino en el encabezado de Ethernet con la dirección MAC de destino en el cuerpo ARP:

(config)# ip arp inspection validate dst-mac

3. Por la ip - comprueba el cuerpo ARP en busca de direcciones IP no válidas e inesperadas, incluidas las direcciones 0.0.0.0, 255.255.255.255 y todas las direcciones de multidifusión IP:

(config)# ip arp inspection validate ip
