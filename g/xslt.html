¿Para qué sirve XSLT?

XSLT es un lenguaje estandarizado por W3C para transformar documentos XML a otros formatos: HTML, PDF, CSV, etc.

Esta transformación en necesaria ya que XML solo guarda los datos y no se puede usar para leer ni imprimir sin antes haberlo transformado a un formato válido.

En esta transformación se usan las siguientes tecnologías:

  • XPATH: Para navegar entre los distintos elementos del documento XML.
  • XSLT: Para definir el modo de transformación del documento XML.
  • XSL-FO: Permite transformar un documento XML en otro documento que tenga un formato legible e imprimible.

Vincular el documento XML

Primero se deberá vincular el documento XML a la hoja XSL. Para ello hay que añadir la siguiente línea en el prólogo del documento XML: 

<?xml-stylesheet type="text/xsl" href="transformacion.xsl"?>

Reemplace "transformacion.xsl" por la ruta relativa o absoluta de su hoja XSL.

Estructura XSLT

Las reglas de transformación XSLT se denominan hojas XSL almacenadas en ficheros de texto plano con extensión .xsl.

Tipos de elementos:

  • Elementos XSLT: Definen las reglas de la transformación.
  • Elementos LRE: Elementos literales que se replican en la transformación.
  • Elementos de extensión: Son manejados por implementaciones concretas del procesador.

Dentro de los Elementos XSLT hay:

  • Elemento raíz: Es el elemento stylesheet o transform.
  • Elementos de nivel superior: Son hijos directos del elemento raíz.
  • Instrucciones: Están contenidas dentro de elementos de nivel superior.

Elemento raíz

El elemento stylesheet engloba a todos los elementos de la hoja XSL:

<?xml version="1.0" encoding="UTF-8"?>

<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <!-- Aquí se pondrán los elementos XSLT -->

</xsl:stylesheet>
Atributos del elemento raíz - stylesheet 
version=""
 Es importante indicar que sea "2.0". 
xmlns:xsl=""
 Es la declaración de un espacio de nombres "xsl" para el elemento stylesheet y normalmente se declara en la misma URL.

El elemento transform es alternativo al otro elemento raíz stylesheet:

<?xml version="1.0" encoding="UTF-8"?>

<xsl:transform version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <!-- Aquí se pondrán los elementos XSLT -->

</xsl:transform>
Atributos del elemento raíz - transform 
version=""
 Es importante indicar que sea "2.0". 
xmlns:xsl=""
 Es la declaración de un espacio de nombres "xsl" para el elemento transform y normalmente se declara en la misma URL.

Importante: Si el elemento raíz está vacío (o solo tiene el elemento xsl:output dentro), el resultado sería todos los elementos del documento XML.

Elemento <xsl:output/>

Este elemento permite definir el formato de salida de la transformación XSLT. Se suele situar al principio dentro del elemento raíz:

<xsl:output
    method="xml|html|text"        indent="yes|no"

    version=""                    encoding=""
    omit-xml-declaration="yes|no" standalone="yes|no"
    doctype-public=""             doctype-system=""
    cdata-section-elements=""     media-type=""
/>
Atributos del elemento - output 
method="xml|html|text"
 El formato de salida de la transformación. Por defecto es "xml". 
indent="yes|no"
 Si la salida está indentada con su estructura de salida (XML, HTML). 
version=""
 Si la salida es XML o HTML se puede especificar qué versión usar. 
encoding=""
 Codificación de carácteres de la transformación del fichero. 
omit-xml-declaration="yes|no"
 Indica si la declaración XML puede ser omitida o debe ser incluída. Por defecto es "no". 
standalone="yes|no"
 Indica si la declaración del standalone puede aparecer. 
doctype-public=""
 Permite establecer el valor del atributo PUBLIC en la declaración del DOCTYPE del fichero de salida. 
doctype-system=""
 Permite establecer el valor del atributo SYSTEM en la declaración del DOCTYPE del fichero de salida. 
cdata-section-elements=""
 Permite separar, mediante espacios en blanco, los elementos de una lista cuyo contenido debería ser escrito como secciones CDATA. 
media-type=""
 Permite definir el tipo MIME del fichero de salida. Por defecto es "text/xml".

Elemento <xsl:template>

Este elemento es fundamental ya que permite definir una plantilla que se aplicará al contenido a transformar:

<xsl:template match="expresion-XPATH" name="nombre_plantilla">
    
    <!-- Contenido de la plantilla -->

</xsl:template>
Atributos del elemento - template 
match=""
 Es la expresión de XPATH que selecciona los elementos XML al que aplicar la plantilla. 
name=""
 Nombre personalizado para la plantilla para poder invocarlo mediante xsl:call-template.

Usos de match="":

  • Para seleccionar el elemento raíz del documento - /.
  • Para seleccionar un varios elementos específicos del documento - nombre_elemento.
  • Más selecciones en la guía de XPATH.

Elemento <xsl:template> - Ejemplo de transformación en HTML

Dentro de la plantilla se puede tener la siguiente estructura HTML básica, donde podremos modificar el contenido dentro de la etiqueta <body>:

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:output method="html" indent="yes" />

    <xsl:template match="/">
        <!-- Inicio de la plantilla -->

        <html>
            <head>
                <title>Título del documento transformado</title>
            </head>

            <body>
            
                <!-- Contenido del documento HTML transformado -->

            </body>
        </html>

        <!-- Fin de la plantilla -->
    </xsl:template>

</xsl:stylesheet>

Elemento <xsl:template> - Ejemplo de transformación en XML

Dentro de la plantilla se puede tener la siguiente estructura XML básica, donde podremos modificar el contenido dentro de la etiqueta raíz:

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:output method="xml" indent="yes" />

    <xsl:template match="/">
        <!-- Inicio de la plantilla -->

        <etiqueta_xml_raiz>

            <!-- Contenido del documento XML transformado -->

        </etiqueta_xml_raiz>

        <!-- Fin de la plantilla -->
    </xsl:template>

</xsl:stylesheet>

Elemento <xsl:template> - Ejemplo de transformación en TEXT

Dentro de la plantilla se puede tener la siguiente estructura, donde podremos modificar el contenido dentro del template según las necesidades:

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:output method="text" />

    <xsl:template match="/">
        <!-- Inicio de la plantilla -->

        <!-- Contenido del documento de texto transformado -->

        <!-- Fin de la plantilla -->
    </xsl:template>

</xsl:stylesheet>

Elemento <xsl:call-template>

Este elemento permite invocar a un template definido previamente mediante su nombre definido:

<xsl:call-template name="nombre-plantilla" />
Atributos del elemento - call-template 
name=""
 Nombre de la plantilla que se va a invocar.

Elemento <xsl:call-template> - Ejemplo

A continuación se muestra un ejemplo de uso del elemento <xsl:call-template>:

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:output method="html" indent="yes" />

    <xsl:template name="plantilla-parrafo">
        <p>Texto del párrafo</p>
    </xsl:template>

    <xsl:template match="/">
        <html>
            <head>
                <title>Título del documento transformado</title>
            </head>

            <body>
            
                <xsl:call-template name="plantilla-parrafo" />
                <xsl:call-template name="plantilla-parrafo" />

            </body>
        </html>

    </xsl:template>

</xsl:stylesheet>

Como resultado daría el siguiente documento HTML:

<!DOCTYPE html>
<html>
    <head>
        <title>Título del documento transformado</title>
    </head>

    <body>
    
        <p>Texto del párrafo</p>
        <p>Texto del párrafo</p>

    </body>
</html>

Elemento <xsl:value-of>

Este elemento permite coger un valor mediante consultas XPATH:

<xsl:value-of select="expresion-xpath" />
Atributos del elemento - value-of 
select=""
 Es una consulta en XPATH que permite coger valores del documento XML a transformar.

Estas consultas son relativas a donde se sitúe este elemento:

  • Si está dentro del elemento xsl:template, este define mediante su atributo match="" donde se empiezan a realizar las consultas.
  • Si está dentro del elemento xsl:for-each, este define mediante su atributo select="" donde se empiezan a realizar las consultas.

Elemento <xsl:value-of> - Ejemplo

A continuación se muestra un ejemplo de uso del elemento <xsl:value-of>:

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:output method="html" indent="yes" />

    <xsl:template match="/">
        <xsl:value-of select="informacion/titulo" />
    </xsl:template>

</xsl:stylesheet>

Como resultado daría el siguiente documento HTML, transformando el párrafo al contenido que haya dentro del documento XML que coincida con la expresión XPATH /informacion/titulo:

Elemento <xsl:value-of> - Ejemplo con varios elementos

A continuación se muestra un ejemplo de uso del elemento <xsl:value-of> pero cogiendo valores de elementos de un elemento común repetido múltiples veces en el documento XML:

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:output method="text" />

    <xsl:template match="persona">
        <xsl:value-of select="informacion/nombre" />,<xsl:value-of select="informacion/apellido" />
    </xsl:template>

</xsl:stylesheet>

Como resultado daría el siguiente documento de texto, como resultado el "nombre" y "apellido" de cada elemento "persona" separados por una coma:

nombre1,apellido1
nombre2,apellido2
...

Otra forma de hacerlo es seleccionando con el elemento xsl:template a / y usando el elemento xsl:for-each.
