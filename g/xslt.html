¿Para qué sirve XSLT?

XSLT es un lenguaje estandarizado por W3C para transformar documentos XML a otros formatos: HTML, PDF, CSV, etc.

Esta transformación en necesaria ya que XML solo guarda los datos y no se puede usar para leer ni imprimir sin antes haberlo transformado a un formato válido.

En esta transformación se usan las siguientes tecnologías:

  • XPATH: Para navegar entre los distintos elementos del documento XML.
  • XSLT: Para definir el modo de transformación del documento XML.
  • XSL-FO: Permite transformar un documento XML en otro documento que tenga un formato legible e imprimible.

Vincular el documento XML

Primero se deberá vincular el documento XML a la hoja XSL. Para ello hay que añadir la siguiente línea en el prólogo del documento XML: 

<?xml-stylesheet type="text/xsl" href="transformacion.xsl"?>

Reemplace "transformacion.xsl" por la ruta relativa o absoluta de su hoja XSL.

Estructura XSLT

Las reglas de transformación XSLT se denominan hojas XSL almacenadas en ficheros de texto plano con extensión .xsl.

Tipos de elementos:

  • Elementos XSLT: Definen las reglas de la transformación.
  • Elementos LRE: Elementos literales que se replican en la transformación.
  • Elementos de extensión: Son manejados por implementaciones concretas del procesador.

Dentro de los Elementos XSLT hay:

  • Elemento raíz: Es el elemento stylesheet o transform.
  • Elementos de nivel superior: Son hijos directos del elemento raíz.
  • Instrucciones: Están contenidas dentro de elementos de nivel superior.

Importante - En cualquier expresión XPATH es recomendable usar entidades predefinidas como &gt; en vez de > para evitar problemas.

Elemento raíz

El elemento stylesheet engloba a todos los elementos de la hoja XSL:

<?xml version="1.0" encoding="UTF-8"?>

<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <!-- Aquí se pondrán los elementos XSLT -->

</xsl:stylesheet>
Atributos del elemento raíz - stylesheet 
version=""
 Es importante indicar que sea "2.0". 
xmlns:xsl=""
 Es la declaración de un espacio de nombres "xsl" para el elemento stylesheet y normalmente se declara en la misma URL.

El elemento transform es alternativo al otro elemento raíz stylesheet:

<?xml version="1.0" encoding="UTF-8"?>

<xsl:transform version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <!-- Aquí se pondrán los elementos XSLT -->

</xsl:transform>
Atributos del elemento raíz - transform 
version=""
 Es importante indicar que sea "2.0". 
xmlns:xsl=""
 Es la declaración de un espacio de nombres "xsl" para el elemento transform y normalmente se declara en la misma URL.

Importante: Si el elemento raíz está vacío (o solo tiene el elemento xsl:output dentro), el resultado sería todos los elementos del documento XML.

Elemento <xsl:output/>

Este elemento permite definir el formato de salida de la transformación XSLT. Se suele situar al principio dentro del elemento raíz:

<xsl:output
    method="xml|html|text"        indent="yes|no"

    version=""                    encoding=""
    omit-xml-declaration="yes|no" standalone="yes|no"
    doctype-public=""             doctype-system=""
    cdata-section-elements=""     media-type=""
/>
Atributos del elemento - output 
method="xml|html|text"
 El formato de salida de la transformación. Por defecto es "xml". 
indent="yes|no"
 Si la salida está indentada con su estructura de salida (XML, HTML). 
version=""
 Si la salida es XML o HTML se puede especificar qué versión usar. 
encoding=""
 Codificación de carácteres de la transformación del fichero. 
omit-xml-declaration="yes|no"
 Indica si la declaración XML puede ser omitida o debe ser incluída. Por defecto es "no". 
standalone="yes|no"
 Indica si la declaración del standalone puede aparecer. 
doctype-public=""
 Permite establecer el valor del atributo PUBLIC en la declaración del DOCTYPE del fichero de salida. 
doctype-system=""
 Permite establecer el valor del atributo SYSTEM en la declaración del DOCTYPE del fichero de salida. 
cdata-section-elements=""
 Permite separar, mediante espacios en blanco, los elementos de una lista cuyo contenido debería ser escrito como secciones CDATA. 
media-type=""
 Permite definir el tipo MIME del fichero de salida. Por defecto es "text/xml".

Elemento <xsl:template>

Este elemento es fundamental ya que permite definir una plantilla que se aplicará al contenido a transformar:

<xsl:template match="expresion-XPATH" name="nombre_plantilla">
    
    <!-- Contenido de la plantilla -->

</xsl:template>
Atributos del elemento - template 
match=""
 Es la expresión de XPATH que selecciona los elementos XML al que aplicar la plantilla. 
name=""
 Nombre personalizado para la plantilla para poder invocarlo mediante xsl:call-template.

Usos de match="":

  • Para seleccionar el elemento raíz del documento - /.
  • Para seleccionar un varios elementos específicos del documento - nombre_elemento.
  • Más selecciones en la guía de XPATH.

Elemento <xsl:template> - Ejemplo de transformación en HTML

Dentro de la plantilla se puede tener la siguiente estructura HTML básica, donde podremos modificar el contenido dentro de la etiqueta <body>:

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:output method="html" indent="yes" />

    <xsl:template match="/">
        <!-- Inicio de la plantilla -->

        <html>
            <head>
                <title>Título del documento transformado</title>
            </head>

            <body>
            
                <!-- Contenido del documento HTML transformado -->

            </body>
        </html>

        <!-- Fin de la plantilla -->
    </xsl:template>

</xsl:stylesheet>

Elemento <xsl:template> - Ejemplo de transformación en XML

Dentro de la plantilla se puede tener la siguiente estructura XML básica, donde podremos modificar el contenido dentro de la etiqueta raíz:

<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:output method="xml" indent="yes" />

        <!-- Inicio de la plantilla -->

        <etiqueta_xml_raiz>

            <!-- Contenido del documento XML transformado -->

        </etiqueta_xml_raiz>

        <!-- Fin de la plantilla -->
    </xsl:template>

</xsl:stylesheet>

<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:output method="text" />

    <xsl:template match="/">
        <!-- Inicio de la plantilla -->

        <!-- Contenido del documento de texto transformado -->

        <!-- Fin de la plantilla -->
    </xsl:template>

</xsl:stylesheet>

Elemento <xsl:call-template>

Este elemento permite invocar a un template definido previamente mediante su nombre definido:

<xsl:call-template name="nombre-plantilla" />
Atributos del elemento - call-template 
name=""
 Nombre de la plantilla que se va a invocar.

Elemento <xsl:call-template> - Ejemplo

A continuación se muestra un ejemplo de uso del elemento xsl:call-template:

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:output method="html" indent="yes" />

    <xsl:template name="plantilla-parrafo">
        <p>Texto del párrafo</p>
    </xsl:template>

    <xsl:template match="/">
        <html>
            <head>
                <title>Título del documento transformado</title>
            </head>

            <body>
            
                <xsl:call-template name="plantilla-parrafo" />
                <xsl:call-template name="plantilla-parrafo" />

            </body>
        </html>

    </xsl:template>

</xsl:stylesheet>

Como resultado daría el siguiente documento HTML:

<!DOCTYPE html>
<html>
    <head>
        <title>Título del documento transformado</title>
    </head>

    <body>
    
        <p>Texto del párrafo</p>
        <p>Texto del párrafo</p>

    </body>
</html>

Elemento <xsl:value-of>

Este elemento permite coger un valor mediante consultas XPATH:

<xsl:value-of select="expresion-xpath" />
Atributos del elemento - value-of 
select=""
 Es una consulta en XPATH que permite coger varios nodos del documento XML para aplicar el template.

Estas consultas son relativas a donde se sitúe este elemento:

  • Si está dentro del elemento xsl:template, este define mediante su atributo match="" donde se empiezan a realizar las consultas.
  • Si está dentro del elemento xsl:for-each, este define mediante su atributo select="" donde se empiezan a realizar las consultas.

Elemento <xsl:value-of> - Ejemplo

A continuación se muestra un ejemplo de uso del elemento xsl:value-of:

<?xml version="1.0" encoding="UTF-8"?>

    <xsl:output method="html" indent="yes" />

    <xsl:template match="/">
        <xsl:value-of select="informacion/titulo" />
    </xsl:template>

</xsl:stylesheet>

Como resultado daría el siguiente documento HTML, transformando el párrafo al contenido que haya dentro del documento XML que coincida con la expresión XPATH /informacion/titulo:

Elemento <xsl:value-of> - Ejemplo con varios elementos

A continuación se muestra un ejemplo de uso del elemento xsl:value-of pero cogiendo valores de elementos de un elemento común repetido múltiples veces en el documento XML:

<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:output method="text" />

    <xsl:template match="persona">
        <xsl:value-of select="informacion/nombre" />,<xsl:value-of select="informacion/apellido" />
    </xsl:template>

</xsl:stylesheet>

Como resultado daría el siguiente documento de texto, como resultado el "nombre" y "apellido" de cada elemento "persona" separados por una coma:

nombre1,apellido1
nombre2,apellido2
...

Otra forma de hacerlo es seleccionando con el elemento xsl:template a / y usando el elemento xsl:for-each.

Elemento <xsl:apply-templates>

El siguiente elemento permite seleccionar a un conjunto de nodos del documento XML y aplicarle un template:

<xsl:apply-templates select="expresion-xpath" />
Atributos del elemento - apply-templates 
select=""
 Es una consulta en XPATH que permite coger varios nodos del documento XML para aplicar el template.

Procesamiento de templates

  1. El procesador XSLT comienza en el nodo raíz del documento.
  2. Busca un xsl:template cuyo atributo match coincida con ese nodo.
    • Si encuentra uno, lo ejecuta.
    • Si no encuentra ninguno:
      • puede aplicar reglas por defecto (seguir bajando por el XML),
      • o no generar salida visible.
  3. Cuando un xsl:template se ejecuta, puede contener un elemento xsl:apply-templates.
    • Esto hace que se seleccionen nuevos nodos y para cada uno: se busque el template correspondiente y se ejecute.
  4. Este proceso se repite.

Elemento <xsl:apply-templates> - Ejemplo

Partiendo del siguiente XML podemos llamar a un template por cada libro:

<biblioteca>
  <libro>
    <titulo>Don Quijote</titulo>
    <autor>Cervantes</autor>
  </libro>
  <libro>
    <titulo>1984</titulo>
    <autor>George Orwell</autor>
  </libro>
</biblioteca>
<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:template match="/biblioteca">
        <html>
            <body>
                <h2>Listado de libros</h2>

                <xsl:apply-templates select="libro" />

            </body>
        </html>
    </xsl:template>

    <!-- Contenido del template para cada libro -->
    <xsl:template match="libro">
        <div>
            <h3> <xsl:value-of select="titulo" /> </h3>
            <p>Autor: <xsl:value-of select="autor" /></p>
        </div>
    </xsl:template>

</xsl:stylesheet>

Elemento <xsl:for-each>

El siguiente elemento permite crear un bucle que aplicará su contenido por cada nodo seleccionado:

<xsl:for-each select="expresion-xpath">
    
    <!-- Contenido del template que se aplicará a cada nodo -->
    
</xsl:for-each>
Atributos del elemento - for-each 
select=""
 Es una consulta en XPATH que permite coger varios nodos del documento XML para aplicar el template.

Elemento <xsl:for-each> - Ejemplo

Por cada nodo "libro" dentro del nodo "biblioteca" se le aplicará el contenido dentro de la etiqueta xsl:for-each:

<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:template match="/biblioteca">
        <html>
            <body>
                <h2>Listado de libros</h2>

                <xsl:for-each select="libro">
                    <!-- Inicio del bucle -->

                    <div>
                        <h3> <xsl:value-of select="titulo" /> </h3>
                        <p>Autor: <xsl:value-of select="autor" /></p>
                    </div>

                    <!-- Fin del bucle (para este libro) -->
                </xsl:for-each>

            </body>
        </html>
    </xsl:template>

</xsl:stylesheet>

Elemento <xsl:sort>

El siguiente elemento se sitúa dentro de un xsl:for-each para ordenar los nodos según unas opciones a configurar:

<xsl:sort
    select="expresion-xpath"
    order="ascending|descending"
    case-order="upper-first|lower-first"
    data-type="string|number"
/>
Atributos del elemento - sort 
select=""
 Es una consulta en XPATH que permite seleccionar un texto o valor de un nodo. 
order=""
 Permite confiurar si el orden es ascendiente o descendiente. Por defecto es ascending (ascendiente). 
case-order=""
 Si hay varias coincidencias se puede configurar que las mayúsculas se ordenen primero o las minúsculas. Por defecto es upper-first (mayúsculas primero). 
data-type=""
 Se define el tipo de valor que contiene el nodo para poder filtrarlo. Por defecto es string.

Elemento <xsl:sort> - Ejemplo

Dentro del xsl:for-each se ordenará cada nodo "libro" según su título en formato de texto en orden descendiente:

<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:template match="/biblioteca">
        <html>
            <body>
                <h2>Listado de libros</h2>

                <xsl:for-each select="libro">

                    <xsl:sort select="titulo" order="descending" data-type="string" />

                    <div>
                        <h3> <xsl:value-of select="titulo" /> </h3>
                        <p>Autor: <xsl:value-of select="autor" /></p>
                    </div>

                </xsl:for-each>

            </body>
        </html>
    </xsl:template>

</xsl:stylesheet>

Elemento <xsl:attribute>

El siguiente elemento permite crear en un XML o HTML de salida un atributo, definiendo su nombre y el valor que tendrá:

<xsl:attribute name="nombre">
    <xsl:value-of select="expresion-xpath" />
</xsl:attribute>
Atributos del elemento - attribute 
name=""
 Es el nombre que tendrá el atributo dentro de un nodo.
Atributos del elemento - value-of 
select=""
 Es una consulta en XPATH que permite coger el valor de un nodo que tendrá este atributo.

Elemento <xsl:attribute> - Ejemplo

Dentro de cada nodo "libro" definiré el nombre del autor como atributo "nombre_autor":

<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:template match="/biblioteca">
        <biblioteca>
            <titulo>Listado de libros</titulo>

            <xsl:for-each select="libro">

                <libro>
                    <xsl:attribute name="nombre_autor">
                        <xsl:value-of select="autor" />
                    </xsl:attribute>

                    <titulo> <xsl:value-of select="titulo" /> </titulo>
                </libro>

            </xsl:for-each>
        </biblioteca>
    </xsl:template>

</xsl:stylesheet>

Alternativa al elemento <xsl:attribute>

La siguiente sintaxis permite escribir el atributo a mano seleccionando un valor con XPATH:

<etiqueta_xml nombre_atributo="{expresion-xpath}" />
Sintaxis 
etiqueta_xml
 Hay que seleccionar el nodo XML al que se le aplicará el atributo. 
nombre_atributo=""
 Se define el nombre del atributo a crear y, entre comillas se define la consulta XPATH que se usará como valor. Debe estar entre {}.

Para poner el conjunto de atributos en un elemento XML se debe usar el siguiente atributo:

Sintaxis

use-attribute-sets=""

Elemento <xsl:if>

Este elemento permite crear una condicional para interpretar algo solo si una condición se cumple:

<xsl:if select="expresion-xpath">
    
    <!-- Contenido a interpretar si la condición es verdadera -->
    
</xsl:if>
Atributos del elemento - if 
select=""
 Es una consulta en XPATH que permite filtrar si algo cumple una condición.

Elemento <xsl:if> - Ejemplo

En el siguiente ejemplo solo se enseñan los libros cuyo autor empieze con la letra "A":

<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:template match="/biblioteca">
        <biblioteca>
            <titulo>Listado de libros</titulo>

            <xsl:for-each select="libro">

                <xsl:if select="starts-with(autor, 'A')">

                    <libro>
                        <titulo> <xsl:value-of select="titulo" /> </titulo>
                    </libro>

                </xsl:if>

            </xsl:for-each>
        </biblioteca>
    </xsl:template>

</xsl:stylesheet>

Elemento <xsl:choose>

Este elemento permite crear condicionales para interpretar algo en función de que cumple la condición:

<xsl:choose>
    <xsl:when test="condicion">
        <!-- Qué se interpreta si la condición es verdadera -->
    </xsl:when>

    <!-- Se puede poner todos los elementos <xsl:when> que se quieran aquí -->

    <xsl:otherwise>
        <!-- Qué se interpreta si ninguna condición anterior se cumple - Es opcional -->
    </xsl:otherwise>
    
</xsl:choose>
Atributos del elemento - when 
test=""
 Es una consulta en XPATH que permite filtrar si algo cumple una condición.

Elemento <xsl:choose> - Ejemplo

El siguiente código crea una etiqueta "color" dentro de "libro" y su valor es variable según el "precio" que tenga este "libro":

  1. Si el precio es 0 - el color es rojo.
  2. Si el precio es 10 - el color es naranja.
  3. Si el precio es mayor a 20 - el color es verde.
  4. Si no cumple ninguna condición anterior - el color es negro.
<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:template match="/biblioteca">
        <biblioteca>
            <titulo>Listado de libros</titulo>

            <xsl:for-each select="libro">

                <xsl:choose>

                    <xsl:when test="precio = 0">
                        <color>rojo</color>
                    </xsl:when>
                    <xsl:when test="precio = 10">
                        <color>naranja</color>
                    </xsl:when>
                    <xsl:when test="precio > 20">
                        <color>verde</color>
                    </xsl:when>

                    <xsl:otherwise>
                        <color>negro</color>
                    </xsl:otherwise>

                </xsl:choose>

            </xsl:for-each>
        </biblioteca>
    </xsl:template>

</xsl:stylesheet>

Elemento <xsl:variable>

Este elemento permite almacenar valores para ser usados más tarde en otras partes del documento:

<xsl:variable name="nombre" select="expresion-xpath">

    <!-- Aquí se definirá que valor guardará la variable -->
    
</xsl:variable>
Atributos del elemento - variable 
name=""
 Nombre identificativo para poder llamar a esta variable más tarde. 
select=""
 Es una consulta en XPATH que permite guardar en la variable el valor que de como resultado esta consulta. Es opcional.

Resumen - Puedes guardar un valor estático definiendolo dentro de esta etiqueta xsl:variable o coger el valor desde una consulta XPATH.

Tipos de variables:

  • Global - Su contenido no se podrá modificar. Se declara fuera de un elemento xsl:template.
  • Local - Su contenido se puede modificar. Se declara dentro de un elemento xsl:template.

Elemento <xsl:variable> - Coger su contenido con <xsl:copy-of>

En cualquier lugar del documento XSLT donde se quiera coge rel contenido de la variable, solo hay que poner:

<xsl:copy-of select="$nombre" />
Atributos del elemento - copy-of 
select=""
 Es el nombre identificativo de la variable a la que se hace referencia. Es importante poner el $ al principio.

Otra forma simple es usar solo esto:

$nombre

Elemento <xsl:variable> - Ejemplo

Con el siguiente código se crea una variable cuyo valor es un estilo CSS y a este se le aplicará a todas las etiquetas:

<xsl:stylesheet version="2.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

    <xsl:template match="/biblioteca">
        <html>
            <body>
                <h2>Listado de libros</h2>

                <xsl:for-each select="libro">

                    <div style="{$estilo_css}">

                        <h3> <xsl:value-of select="titulo" /> </h3>
                        <p>Autor: <xsl:value-of select="autor" /></p>
                    </div>

                </xsl:for-each>

            </body>
        </html>
    </xsl:template>

    <!-- Esta es una variable global -->
    <xsl:variable name="estilo_css">
        border: 1px solid red; background-color: white;
    </xsl:variable>

</xsl:stylesheet>
