Iptables

Filtrado de paquetes

Teoría - Filtrado

El filtrado de paquetes es una técnica usada por seguridad y para mejorar el rendimiento que permite analizar el tráfico de red al atravesar un dispositivo y tomar decisiones acerca del mismo pudiendo permitir o denegarlo, modificar el tráfico o separarlo.

Niveles de filtrado:

  • A nivel de enlace: MAC.
  • A nivel de red: IP.
  • A nivel de transporte: Puertos, estado de la conexión.
  • A nivel de aplicación: Tipos de paquetes, DNS.

Seguimiento de la conexión: También puede filtrarse paquetes dependiendo si está relacionado o no con un paquete anterior. Por ejemplo: Bloquear peticiones DNS hacia una red que previamente no realizó ninguna consulta DNS. Se hace manteniendo el estado de las conexiones previas.

Sistemas de filtrado:

  • iptables: Sucesor de ipfwadm e ipchains. No se desarrollan ya nuevas funcionalidades. Ampliamente utilizado. Disponible desde linux 2.4. Desarrollado por netfilter. Funciona solo sobre kérnel Linux en todas las distribuciones, ya viene preinstalado.
  • nftables: Sucesor de iptables. Disponible desde linux 3.13. Desarrollado por netfilter. Alto rendimiento. Se da más peso al espacio de usuario.
  • bpfilter: Permite usar bpf en linux. Disponible desde linux 4.18. La más reciente. Alto rendimiento.

Teoría - Firewall

Firewall: Sistema de seguridad que inspecciona el tráfico de red y selecciona el tráfico mediante reglas. Las apps de filtrado de paquetes se utilizan como firewalls.

Pueden ser dispositivos hardware específicos (Network Appliances) o programas software. Un dispositivo que los implementa puede además actuar como proxy, vpn, dispositivo NAT y router peor un firewall es estrictamente que actúa como control del tráfico.

NAT (Network Address Translation) - Utilizado para:

  • SNAT (Source NAT): Permite traducir una IP de una LAN a una IP pública, por ejemplo: acceder a internet. También puede usarse para evitar añadir reglas de enrutamiento estático.
  • DNAT (Destination NAT): Sirva para exponer un dispositivo de una red local para que sea accesible desde internet. Se usa mediante redirección de puertos.

Categorías de un Firewall:

  • Por nodo: Ubicados en cada nodo de la red protegiéndolo individuamente. Actúa en tráfico interno y externo.
  • Perimetral: Ubicados en un nodo perimetral de la red que protegen a toda la red. Se usa en ámbito corporativo.

Políticas de un firewall:

  • Política DROP: No se permite lo que esté implícitamente permitido en una regla.
  • Política ACCEPT: Acepta todo por defecto a no ser que se deniegue por una regla.

Teoría - DMZ

La DMZ (Zona Desmilitarizada) es una red separada física o lógicamente en la que se ubican los servidores que deben ser accesibles desde otra red

Teoría - Iptables

Elementos de Iptables:

Flujo de tráfico: Se puede clasificar por entrada al nodo, salida del nodo y atravesar el nodo por interfaces.

Jerarquía: Hay reglas que se agrupan en cadenas y estas se agrupan en tablas.

Tablas:

  • filter - Para hacer firewall. Cadenas:
    • INPUT - Entrada al nodo
    • OUTPUT - Salida del nodo
    • FORWARD - Reenvío en el nodo
  • nat - Para hacer NAT. Cadenas:
    • PREROUTING - Para modificar cadenas de antes del reenvio de un paquete (antes de todo)
    • INPUT
    • OUTPUT
    • POSTROUTING - Para modificar cadenas después del reenvío del paquete (después de todo)
  • mangle - Por paquetes.
  • raw - Depurar seguimiento de la conexión.
  • security - Por MAC.

Tabla FILTER:

Tabla FILTER y NAT (INPUT es tanto de filter y nat; postrouting es de nat):

Acciones (TARGETs): Las reglas de iptables fincional como ACL (Listas de Control de Acceso) leyéndose secuencialmente hasta que se aplique una regla. Cuando se defina una regla hay que determinar si es ACCEPT (El paquete pasa), DROP (Se elimina el paquete) o RETURN (No leer más reglas y pasar a la siguiente cadena).

Comando

Todas las reglas se aplicarán con el siguiente comando seguido de unos parámetros:

iptables ...

Establecer políticas

Esta es la sintaxis para establecer una política:

iptables <-t TABLA> -P <CADENA> ACCEPT/DROP

-t: Tabla a modificar. Por defecto es la tabla "filter".

-P: Establecer la política.

ACCEPT/DROP: Es la política que se va a emplear - ACCEPT (Aceptar todo menos lo configurado), DROP (Denegar todo menos lo configurado).

Ejemplos:

iptables -P INPUT DROP
iptables -P OUTPUT ACCEPT

Listar políticas

Esta es la primera opción para listar políticas aplicadas:

iptables [-t TABLA] -L [CADENA] [-n] [-v] [--line-numbers]

-t: Tabla a modificar. Por defecto es la tabla "filter". Se separan por « Chain INPUT », « Chain OUTPUT », « Chain FORWARD ».

-L: Listar.

CADENA: INPUT, OUTPUT, FORWARD.

-n: De forma numérica: no resolver IPs a nombre.

-v: De forma detallada.

--line-numbers: Añade el número de línea por conjunto.

Ejemplo:

iptables -L -nv --line-numbers

Hay otra manera para listarlo pero como se introdujo en la tabla con el siguiente comando:

iptables -S [CADENA] [-v]

-S: Devuelve la cadena con el formato que se introdujo en la tabla.

CADENA: INPUT, OUTPUT, FORWARD.

-v: De forma detallada.

Ejemplo:

iptables -S -v

Añadir reglas

Esta es la sintaxis para crear una regla:

iptables (-A o -I) [CADENA] [-p PROTOCOLO] [-s IP_ORIGEN] [-d IP_DESTINO] [-i INTERFAZ_ENTRADA] [-o INTERFAZ_SALIDA] [--sport PUERTO_ORIGEN] [--dport PUERTO_DESTINO] [-j ACCEPT/DROP]

-A (APPEND): Los añade al final de la ACL.

-I (INSERT): Los añade al principio de la ACL.

CADENA: INPUT, OUTPUT, FORWARD.

-d -s: Puede ser la IP de un nodo (192.168.1.100) o la de una red (192.168.1.0/24).

-j: Regla de aceptar (ACCEPT) o denegar (DROP).

Para decir los que NO sea algo se pone una exclamación antes de la opción como: ! -p-

Ejemplo:

iptables -A INPUT -i enp0s3 -p icmp -j ACCEPT

Comprobación si existe una regla

A continuación puedes comprobar que una política en específico esté creada:

iptables -C [CADENA] [-p PROTOCOLO] [-s IP_ORIGEN] [-d IP_DESTINO] [-i INTERFAZ_ENTRADA] [-o INTERFAZ_SALIDA] [-j ACCEPT/DROP]

-C (CHECK).

Ejemplo:

iptables -C INPUT -i enp0s3 -p icmp -j ACCEPT

Eliminar una regla

Para eliminar una regla hay que ejecutar:

iptables -D [CADENA] [-p PROTOCOLO] [-s IP_ORIGEN] [-d IP_DESTINO] [-i INTERFAZ_ENTRADA] [-o INTERFAZ_SALIDA] [-j ACCEPT/DROP]

-D (DELETE).

Ejemplo:

iptables -D INPUT 1

El 1 es la primera regla de Chain INPUT.

Eliminar todas las reglas

Para purgar todas las reglas hay que ejecutar:

iptables -F [CADENA]

-F (FLUSH).

CADENA: Si no se indica borra todas las cadenas o solo la especificada.

Reiniciar contadores

Para reiniciar los contadores sin borrar las reglas:

iptables -Z [CADENA]

-Z: No borrar las reglas pero si resetear su contador de políticas.

CADENA: De todas las cadenas por defecto o de la especificada.

Cadenas

INPUT: Paquete recibido por el nodo. Útil para firewall personal, no perimetral.

iptables -A INPUT

OUTPUT: Paquete generado por el nodo. Útil para firewall personal, no perimetral.

iptables -A OUTPUT

FORWARD: Útil para firewall perimetral, no personal.

iptables -A FORWARD

Forward

Paquetes que atraviesan el nodo. Utiliza la tabla filter. Deben permitirse los paquetes en ambos sentidos al ser perimetral.

Para permitir el reenvío de interfaces:

echo 1 > /proc/sys/net/ipv4/ip_forward

SNAT (Source NAT)

Cambia la IP de origen por la externa del dispositivo NAT definiéndose en las cadenas de POSTROUTING de la tabla NAT. Esto se hace si la IP es ESTÁTICA.

Ejemplo:

iptables -t nat -A POSTROUTING -s 192.168.1.0/24 -o eth0 -j SNAT –to 80.14.1.2/32

En este ejemplo se añade en la tabla nat la cadena postrounting (después de procesar todo) a la red local que salga por la IP pública especificada.

Enmascaramiento IP (IP Masquerade): Es cuando la IP del dispositivo origen es DINÁMICA con SNAT. Se debe comprobar la IP externa del dispositivo NAT antes de cambiarla en el paquete.

Ejemplo:

iptables -t nat -A POSTROUTING -s 192.168.1.0/24 -o eth0 -j MASQUERADE

En este ejemplo se mira la IP automáticamente y se cambia.

DNAT (Destination NAT)

Se usa para exponer el puerto de un nodo privado al exterior cambiando la IP destino de la externa del nodo a la del nodo a exponer. Se define en PREROUTING de la tabla NAT.

Para reenviar peticiones a un puerto del dispositivo NAT de un nodo interno:

iptables -t nat -A PREROUTING -p [PROTOCOLO: tcp/udp/..] –dport [PUERTO] -i [INTERFAZ] -j DNAT –to [IP_A_REDIRIGIR]( :OTRO_PUERTO_OPCIONAL)

Extensiones de Iptables

Las extensiones permiten ampliar las funcionalidades de iptables mediante el parámetro -m MÓDULO. Para instalarlas:

sudo apt install iptables-extensions

En todas las extensiones se utiliza el parámetro de acción:

-j REJECT/ACCEPT

Filtros de IP:

Filtrar paquetes según su tipo de dirección:

-m addrtype --dst-type BROADCAST/UNICAST/MULTICAST

Seleccionar rangos de IP:

-m iprange --src-range 192.168.1.1-192.168.1.100 --dst-range 10.0.0.1-10.0.0.50

Comentarios y documentación:

Permite añadir comentarios a las reglas para que aparezcan al listar las reglas:

-m comment --comment "Bloqueo temporal de IPs"

Control de conexiones y tráfico:

Limitar el peso en bytes de un paquete:

-m connbytes --connbytes 1024:4096

Limitar el número de conexiones simultáneas por cliente:

-m connlimit --connlimit-above 3

Seleccionar tráfico ICMP:

-m icmp --icmp-type echo-request/echo-reply

Limitar paquetes por velocidad:

-m limit --limit 10/sec

Seleccionar por MAC de origen:

-m mac --mac-source 00:11:22:33:44:55

Especificar múltiples puertos a la vez:

-m multiport --sports 80,443 --dports 1024:2048

Limitar el número total de paquetes por cadena:

-m quota --quota 50000

Módulos de protocolo:

TCP:

-p tcp --dport 22 --sport 1024:65535 --tcp-flags SYN,RST SYN

UDP:

-p udp --dport 53 --sport 1024:65535

Restricciones temporales:

Aplicar reglas solo en ciertos horarios:

-m time --timestart 08:00 --timestop 18:00

Seguimiento de conexiones (conntrack y state):

Control avanzado del estado de las conexiones:

-m conntrack --ctstate NEW,ESTABLISHED

Filtrar por protocolo de capa 4:

-m conntrack --ctproto tcp/udp

Filtrar por IP de origen/destino y puerto:

-m conntrack --ctorigsrc 192.168.1.5 --ctorigdst 10.0.0.10 --ctorigsrcport 1024

Módulo state como subconjunto de conntrack:

-m state --state NEW,ESTABLISHED

Nuevas cadenas

Se utiliza para organizar las reglas según el tipo de tráfico aumentando así el tráfico. Pej. si ay muchas líneas de iptables y, en vez de leer todas las reglas se puede organizar y controlar.

Crear una nueva cadena:

iptables [-t TABLA] -N NUEVA_CADENA

Definir el criterio para que se lea la nueva cadena:

iptables -A INPUT -s IP_ORIGEN -j NUEVA_CADENA

Con esto se hace un return y solo lee las cadenas de NUEVA_CADENA en vez de poner ACCEPT,DROP. En este caso no hace falta especificar en otras cadenas para NUEVA_CADENA la IP ya que van dirigidas automáticamente para esa IP.

Renombrar una cadena:

iptables [-t TABLA] -E NUEVA_CADENA NUEVO_NOMBRE

Borrar la cadena:

iptables -X NUEVA_CADENA

Creación de scripts

Se puede crear un script de shell donde se definirán todas las reglas y cadenas como el siguiente script de ejemplo:

#!/bin/bash
iptables -F
iptables -t nat -F
iptables -Z
iptables -t nat -Z

iptables -P INPUT DROP
iptables -P OUTPUT DROP
iptables -P FORWARD DROP

# Crear cadenas
# Poner reglas INPUT
# Poner reglas OUTPUT
# Poner reglas FORWARD

Es una buena manera de configurar varios equipos sin escribir las cadenas cada vez, además que se permite una organización con comentarios.

Persistencia de la configuración

Para hacer que las reglas persistan, aunque se reinicie el nodo:

iptables-save > ./rules.v4

Quita automáticamente “iptables” de cada línea.

Restaurar las reglas:

iptables-restore ./rules.v4

Restaurar automáticamente las reglas:

nano /etc/systemd/system/iptables.servce

Cambiar ExecStart a /sbin/iptables-restore ./ruta/rules.v4.
