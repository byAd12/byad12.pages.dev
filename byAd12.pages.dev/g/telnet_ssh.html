TELNET

Teletype Network

Adrián L. G. P.

Instalación y configuración

apt-get update
ufw allow 23

Conectarse a otro equipo 

telnet [IP] (PUERTO)

Pide usuario y contraseña. NO viajan cifrados y pueden ser interceptados.

SSH

Secure Shell


Configuración 

apt install openssh-server
ufw allow 22
systemctl status ssh

Configuración 

Start-Service sshd
Set-Service -Name sshd -StartupType 'Automatic'

Habilitar el puerto 22 en el firewall en reglas de entrada y salida.

Conectarse a otro equipo 

ssh (USER)@[IP]

Pide usuario y contraseña. SI viajan cifrados y NO pueden ser interceptados.

Redireccionar HTTP por un socket al servidor

1. Editar el archivo /etc/ssh/sshd_config en el servidor SSH y añadir/cambiar:

AllowTcpForwarding yes
GatewayPorts yes

2. En el servidor reiniciar el servicio SSH:

systemctl restart sshd

3. Crear un socker por el que transmitir la información:

ssh -C -D 2021 (USER)@[IP]

4. En el navegador del cliente configurar la red a "Configuración manual de proxy", configurando un socker con la ip 127.0.0.1 y el puerto 2021 asignado.

Abrir la GUI de un programa del servidor en el cliente

1. Editar el archivo /etc/ssh/sshd_config en el servidor SSH y añadir/cambiar:

X11Forwarding yes

2. En el servidor reiniciar el servicio SSH:

systemctl restart sshd

3. Conectarse con X11 al servidor:

ssh -X (USER)@[IP]

4. Abrir un programa mediante comandos.
